**Natural Language:**
THEOREM: The union of two polynomial-time decidable languages over an alphabet Σ is itself a polynomial-time decidable language. Formally, if L₁ and L₂ are in P, then L₁ ∪ L₂ is also in P.

PROOF SKETCH: 
1. Recall that a language L is in P if there exists a polynomial-time decidable Turing machine M that decides L.
2. Given L₁ and L₂ are in P, we can construct a Turing machine M' that decides L₁ ∪ L₂ as follows:
   a. On input x, M' simulates the Turing machines for L₁ and L₂ in parallel.
   b. If either machine accepts x, then M' accepts x.
   c. If both machines reject x, then M' rejects x.
3. Since the simulation of two polynomial-time machines in parallel still runs in polynomial time, M' is a polynomial-time decidable Turing machine.
4. Therefore, L₁ ∪ L₂ is in P.

LEAN HINT:
- Imports: import complexity.basic
- Tactics: apply, exact, use, have, simp, intro

DIFFICULTY: medium

**Lean 4 Proof:**
```lean

import complexity.basic

open complexity

theorem union_of_poly_time_decidable
  {Σ : Type} [fintype Σ]
  (L₁ L₂ : language Σ)
  (h₁ : L₁ ∈ P)
  (h₂ : L₂ ∈ P) :
  L₁ ∪ L₂ ∈ P :=
begin
  rcases h₁ with ⟨M₁, hM₁, p₁⟩,
  rcases h₂ with ⟨M₂, hM₂, p₂⟩,
  let M' := machine.parallel M₁ M₂,
  let p' := λ n, max (p₁ n) (p₂ n),
  use M',
  have : M'.eval x = some tt ↔ x ∈ L₁ ∪ L₂,
  { simp [machine.parallel_eval, hM₁, hM₂] },
  use this,
  have : M'.time x ≤ p' (x.length),
  { simp [machine.parallel_time, p₁, p₂] },
  use this,
  apply polynomial_time_bounded,
  apply poly_time_of_big_o,
  exact big_o.max (p₁.is_big_o) (p₂.is_big_o),
end

THEOREM: The composition of two polynomial-time computable functions is itself polynomial-time computable. Formally, if f and g are polynomial-time computable functions, then the function h defined by h(x) = f(g(x)) is also polynomial-time computable.

PROOF SKETCH: 
1. Recall the definition of polynomial-time computable functions: a function f is polynomial-time computable if there exists a polynomial p such that the time complexity of computing f(x) is bounded by p(|x|), where |x| denotes the length of the input x.
2. Let p_f and p_g be the polynomials bounding the time complexities of f and g, respectively.
3. The time complexity of computing h(x) = f(g(x)) involves first computing g(x), which takes time at most p_g(|x|), and then computing f(g(x)), which takes time at most p_f(|g(x)|).
4. Since |g(x)| is the length of the output of g(x), we need to relate this to |x|. Because g is polynomial-time computable, the length of its output |g(x)| is polynomially bounded in |x|, i.e., |g(x)| ≤ q(|x|) for some polynomial q.
5. Substitute this bound into the polynomial for f: p_f(|g(x)|) ≤ p_f(q(|x|)), which is still a polynomial in |x|.
6. Therefore, the total time complexity of h(x) is bounded by p_g(|x|) + p_f(q(|x|)), which is a polynomial in |x|.
7. Hence, h is polynomial-time computable.

  (f g : ℕ → ℕ) (hf : polyTimeComputable f) (hg : polyTimeComputable g) :

  rcases hg with ⟨p_g, hp_g⟩,

**Natural Language:**
THEOREM: The composition of a polynomial-time decidable language with a polynomial-time computable function results in a polynomial-time decidable language. Formally, if L is a polynomial-time decidable language over an alphabet Σ and f: Σ* → Σ* is a polynomial-time computable function, then the language { x | f(x) ∈ L } is also polynomial-time decidable.

PROOF SKETCH:
1. Recall that a language L being polynomial-time decidable means there exists a Turing machine M that decides L in polynomial time.
2. Since f is polynomial-time computable, there exists a Turing machine M_f that computes f in polynomial time.
3. To decide the language { x | f(x) ∈ L }, construct a Turing machine M' that, on input x, first computes f(x) using M_f, then simulates M on f(x).
4. The total running time of M' is the sum of the time to compute f(x) and the time for M to decide f(x), both of which are polynomial in the length of x.
5. Therefore, the constructed language is polynomial-time decidable.

LEAN HINT:
- Import necessary modules: `import complexity.basic`
- Define polynomial-time decidability and computability using existing structures (e.g., `polynomial_time_decidable`, `polynomial_time_computable`).
- Use `exists.intro` to construct the deciding Turing machine M'.
- Apply polynomial addition and composition lemmas to establish the overall polynomial running time.

**Lean 4 Proof:**

import Mathlib
import complexity.basic

def poly_time_decidable (L : Set String) : Prop :=
  ∃ (M : TM0), (∀ x, M.eval x = some (x ∈ L)) ∧ M.poly_time

def poly_time_computable (f : String → String) : Prop :=
  ∃ (M : TM0), (∀ x, M.eval x = some (f x)) ∧ M.poly_time

theorem composition_poly_time_decidable
    (L : Set String) (f : String → String)
    (hL : poly_time_decidable L) (hf : poly_time_computable f) :
    poly_time_decidable { x | f x ∈ L } :=
  begin
    rcases hL with ⟨M, hM, hMp⟩,
    rcases hf with ⟨Mf, hMf, hMfp⟩,
    let M' := Mf.seq M,
    apply exists.intro M',
    split,
    { intro x,
      simp only [TM0.seq_eval, hMf, hM],
      exact eq.refl (M.eval (f x) = some (f x ∈ L)) },
    { apply poly_time.comp,
      { exact poly_time.add hMfp hMp },
      { exact hMfp } }
  end
```
