4. Since |g(x)| is the length of the output of g(x), we need to relate this to |x|. Because g is polynomial-time computable, the length of its output |g(x)| is polynomially bounded in |x|, i.e., |g(x)| ≤ q(|x|) for some polynomial q.

6. Therefore, the total time complexity of h(x) is bounded by p_g(|x|) + p_f(q(|x|)), which is a polynomial in |x|.
7. Hence, h is polynomial-time computable.

  {z | ∃ x y, z = x ++ y ∧ x ∈ L₁ ∧ y ∈ L₂}

  L₁ ∈ P → L₂ ∈ P → L₁ ∪ L₂ ∈ P :=
begin
  intro h₁ h₂,
  obtain ⟨M₁, hM₁, p₁⟩ := h₁,
  obtain ⟨M₂, hM₂, p₂⟩ := h₂,
  let M' := TM.parallel M₁ M₂,
  use M',
  apply TM.accepts_iff.2,
  intro x,
  simp only [TM.parallel_accepts_iff, Language.mem_union],
  exact Or.inl (hM₁ x) || Or.inr (hM₂ x),
  use max p₁ p₂,
  apply TM.parallel_eval_dom,
  simp only [TM.parallel_eval_dom_iff],
  exact (hM₁ x) || (hM₂ x),
  simp only [Polynomial.max, Polynomial.coe_max],
  apply linarith,
  apply le_max_left,
  apply le_max_right,
end
