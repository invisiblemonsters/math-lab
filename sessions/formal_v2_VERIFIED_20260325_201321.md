4. **Polynomial Bound**: Substitute |g(x)| into p_f to get p_f(p_g(|x|)), which is a polynomial in |x| because the composition of polynomials is a polynomial.

THEOREM: The set of polynomial-time decidable languages is closed under the operation of taking the concatenation of two languages, i.e., if languages L₁ and L₂ are in P, then the language L₁ · L₂ = {w₁w₂ | w₁ ∈ L₁ and w₂ ∈ L₂} is also in P.

PROOF SKETCH:
1. **Definition of P**: Recall that a language L is in P if there exists a deterministic Turing machine (DTM) that decides L in polynomial time.
2. **Decidability of L₁ and L₂**: Since L₁ and L₂ are in P, there exist DTMs M₁ and M₂ that decide L₁ and L₂ in polynomial time, respectively.
3. **Constructing M for L₁ · L₂**: We can construct a DTM M that, on input string w, iterates over all possible splits of w into w₁ and w₂, simulates M₁ on w₁ and M₂ on w₂, and accepts if both M₁ and M₂ accept.
4. **Polynomial Time Complexity**: The number of splits is linear in the length of w, and each simulation of M₁ and M₂ runs in polynomial time. Since polynomial time is closed under composition and multiplication by constants, M runs in polynomial time.
5. **Conclusion**: Therefore, L₁ · L₂ is in P.

LEAN HINT:

  unfold language.concat,
  -- Use the decidability of L₁ and L₂ to construct the DTM for L₁ · L₂
  apply exists.intro,
  -- Construct the DTM using the DTMs for L₁ and L₂
  use DTM.concat_decider,
  -- Verify the polynomial time complexity using composition

  -- Apply the given hypotheses for L₁ and L₂

**Natural Language:**
THEOREM: If a language L₂ is polynomial-time reducible to a language L₁ (i.e., L₁ ≤ₚ L₂), and L₁ is in P, then L₂ is also in P.

PROOF SKETCH:
1. **Definition of Reduction**: Recall that a polynomial-time reduction from L₂ to L₁ is a polynomial-time computable function f such that for all x, x ∈ L₂ ⇨ f(x) ∈ L₁.
2. **Decidability of L₁**: Since L₁ is in P, there exists a polynomial-time decidable algorithm for L₁.
3. **Composition**: To decide L₂, compute f(x) and then check if f(x) is in L₁ using the algorithm for L₁. This composition is polynomial-time because both f and the algorithm for L₁ are polynomial-time.
4. **Conclusion**: Therefore, L₂ can be decided in polynomial time, hence L₂ ∈ P.

LEAN HINT:
```lean
import complexity.basic

theorem poly_reduc_closed_P {L₁ L₂ : language} (h₁ : L₁ ∈ P) (h₂ : L₁ ≤ₚ L₂) : L₂ ∈ P :=
begin
  -- Unfold the definitions
  unfold has_le_le_poly at h₂,
  cases h₂ with f h_f,
  -- Use the decidability of L₁
  unfold mem_P at h₁,
  -- Construct the decider for L₂
  use (λx, decidable_of_decidable (h₁ (f x))),
  -- Verify polynomial-time complexity
  apply poly_time_comp h_f h₁,
end
```

**Natural Language:**
THEOREM: If a language L₁ is polynomial-time reducible to a language L₂ (i.e., L₁ ≤ₚ L₂), and L₂ is in P, then the complement of L₁ (denoted as ¬L₁) is also polynomial-time reducible to the complement of L₂ (denoted as ¬L₂).

PROOF SKETCH:
1. **Definition of Reduction**: Recall that L₁ ≤ₚ L₂ means there exists a polynomial-time computable function f such that for all x, x ∈ L₁ ⇨ f(x) ∈ L₂.
2. **Complement Reduction**: To show ¬L₁ ≤ₚ ¬L₂, we need a polynomial-time computable function g such that for all x, x ∈ ¬L₁ ⇨ g(x) ∈ ¬L₂.
3. **Construct g from f**: Observe that the same function f used for the reduction L₁ ≤ₚ L₂ can be used for the complement reduction. Specifically, x ∉ L₁ ⇨ f(x) ∉ L₂, which is equivalent to x ∈ ¬L₁ ⇨ f(x) ∈ ¬L₂.
4. **Polynomial-Time Property**: Since f is polynomial-time computable, g can be taken as f, preserving the polynomial-time property.
