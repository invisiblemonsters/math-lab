THEOREM: The sum of the first n natural numbers, denoted as 1 + 2 + 3 + ... + n, is equal to n(n+1)/2.

PROOF SKETCH: 
1. **Base Case**: Verify the formula for n = 1, where the sum is 1 and the formula yields 1(1+1)/2 = 1.
2. **Inductive Step**: Assume the formula holds for some arbitrary natural number k. Then, for k + 1, show that the sum of the first k + 1 natural numbers equals (k + 1)(k + 2)/2 by adding (k + 1) to both sides of the inductive hypothesis.

LEAN HINT:
- Imports: `import data.nat.basic`
- Tactics: `induction n with n ih`, `simp [nat.add_comm, nat.mul_comm, nat.add_assoc]`, `omega` (if needed for simple arithmetic verification)

DIFFICULTY: easy

**Lean 4 Proof:**
```lean
import Mathlib
import data.nat.basic

theorem sum_of_first_n_natural_numbers (n : ℕ) : 
  (list.range (n + 1)).sum (λ x, x) = n * (n + 1) / 2 :=
begin
  induction n with n ih,
  { simp, },
  { simp [nat.add_comm, nat.mul_comm, nat.add_assoc] at *,
    rw [list.sum_range_succ, ih],
    ring_nf,
    omega, }
end
```

PROOF SKETCH:

THEOREM: For all natural numbers n, the sum of the first n odd numbers is equal to n squared (i.e., 1 + 3 + 5 + ... + (2n-1) = n²).

PROOF SKETCH:
1. **Base Case**: Verify the statement for n = 1, where the sum of the first odd number (1) equals 1².
2. **Inductive Step**: Assume the statement holds for some arbitrary natural number k (i.e., 1 + 3 + 5 + ... + (2k-1) = k²). Then, demonstrate it holds for k + 1 by showing that adding the next odd number (2(k+1)-1 = 2k+1) to both sides of the inductive hypothesis results in (k+1)².

LEAN HINT:
- Imports: `import data.nat.basic`
- Tactics: `induction n with n ih`, `simp [pow_two, mul_add, add_comm]`, `norm_num`

DIFFICULTY: easy

**Lean 4 Proof:**
```lean
import Mathlib
import data.nat.basic

theorem sum_of_odd_numbers (n : ℕ) : 
  (List.range n).map (λ k, 2*k+1) |>.foldl (· + ·) 0 = n^2 :=
begin
  induction n with n ih,
  case zero => 
    simp [List.range, List.map, List.foldl],
  case succ n ih =>

    ring,
```

## Turn 5 — Verified ✓

**Natural Language:**
THEOREM: The composition of two polynomial-time computable functions is itself polynomial-time computable.

PROOF SKETCH:
1. **Definitions**: Let `f` and `g` be polynomial-time computable functions, meaning there exist polynomials `p` and `q` such that the time complexity of `f` is bounded by `p` and `g` is bounded by `q`.
2. **Composition Definition**: Define the composition `h(x) = f(g(x))`.
3. **Time Complexity Analysis**: The time to compute `h(x)` is the time to compute `g(x)` plus the time to compute `f` on the output of `g(x)`. Since `g` runs in time `q(|x|)` and `f` runs in time `p(|g(x)|)`, we need to show that `p(q(|x|))` is a polynomial in `|x|`.
4. **Polynomial Closure**: The composition of two polynomials is a polynomial, thus `p(q(|x|))` is a polynomial in `|x|`, proving that `h` is polynomial-time computable.

LEAN HINT:
```lean
import Complexity.Basic

theorem poly_time_composition {f g : ℕ → ℕ} (hf : poly_time f) (hg : poly_time g) :
  poly_time (λ x, f (g x)) :=
begin
  -- Unfold the definitions of poly_time for f and g
  cases hf, cases hg,
  -- Construct the polynomial for the composition
  have : ∃ p : polynomial, ∀ x, time_complexity (λ x, f (g x)) x ≤ p.eval x,
  { use hf_p ∘ hg_p, -- Compose the polynomials
    intros x,
    -- Time to compute g(x) plus time to compute f(g(x))
    transitivity,
    apply hg_time,
    apply hf_time,
    -- Show the composed polynomial bounds the total time
    simp [polynomial.comp, polynomial.eval], },
  -- Conclude the composition is polynomial-time
  exact ⟨this_p, this_time⟩,
end
```
