**Natural Language:**
THEOREM: The union of two finite sets of natural numbers is also finite.

PROOF SKETCH:
1. **Definition Recall**: A set of natural numbers is finite if it has a finite number of elements.
2. **Union Definition**: The union of two sets A and B, denoted A ∪ B, contains all elements that are in A, in B, or in both.
3. **Finite Union Argument**: If A and B are finite, then the total number of elements in A ∪ B is at most the sum of the number of elements in A and the number of elements in B (by the principle of inclusion-exclusion, but simplified for finite sets).
4. **Conclusion**: Since the sum of two finite numbers is finite, A ∪ B must also be finite.

LEAN HINT:
- Imports: `import data.set.finite`
- Tactics: `intro`, `apply`, `simp`, `exact` (utilizing `set.finite.union` from Mathlib)

DIFFICULTY: easy

**Lean 4 Proof:**
```lean
import Mathlib
import data.set.finite

theorem finite_union {A B : Set ℕ} (hA : A.Finite) (hB : B.Finite) : (A ∪ B).Finite :=
  by exact Set.Finite.union hA hB
```

PROOF SKETCH:

3. Inductive step: Assume the formula holds for some n = k. Then, for n = k + 1, the sum of the first k + 1 natural numbers is k(k+1)/2 + (k + 1). Simplify this expression to show it equals (k + 1)(k + 2)/2, thus proving the inductive step.

- Use `induction n with k ih` to start the induction.

| 0 => 0
| n + 1 => (n + 1) + sum_of_first_n n

PROOF SKETCH:

2. **Composition**: Given two polynomial-time computable functions f and g, we need to show that their composition h(x) = f(g(x)) is also polynomial-time computable.
3. **Turing Machine Construction**: Construct a Turing machine M_h that simulates the computation of g on input x, then feeds the result to the Turing machine for f. 
4. **Time Complexity Analysis**: The running time of M_h is the sum of the running times of the Turing machines for g and f. Since both are polynomial, their sum is also polynomial (because the sum of two polynomials is a polynomial).
5. **Conclusion**: Therefore, h is polynomial-time computable.

## Turn 5 — Verified ✓

**Natural Language:**
THEOREM: The composition of two polynomial functions is itself a polynomial function. Specifically, if f and g are polynomial functions, then the function h(x) = f(g(x)) is also a polynomial function.

PROOF SKETCH: 
1. Recall the definition of a polynomial function: a function f: N → N is polynomial if there exists a polynomial p(n) such that f(n) ≤ p(n) for all n.
2. Assume f and g are polynomial functions with corresponding polynomials p_f(n) and p_g(n), respectively.
3. The composition h(n) = f(g(n)) can be bounded by p_f(p_g(n)), which is a polynomial in n since the composition of polynomials is a polynomial.
4. Therefore, h(n) is a polynomial function.

LEAN HINT:
- Imports: `import data.nat.basic` and `import algebra.polynomial`
- Tactics: Use `apply` with the definition of polynomial functions, `simp` to simplify polynomial expressions, and `ring` to verify polynomial composition.

DIFFICULTY: medium

**Lean 4 Proof:**
```lean
import Mathlib
import data.nat.basic
import algebra.polynomial

-- Define polynomial functions
def is_polynomial (f : ℕ → ℕ) : Prop :=
∃ p : ℕ → ℕ, (∀ n, f n ≤ p n) ∧
             (∃ k : ℕ, ∀ n, p n ≤ n ^ k)

-- Composition of polynomial functions is polynomial
theorem composition_polynomial (f g : ℕ → ℕ) :
  is_polynomial f → is_polynomial g → is_polynomial (f ∘ g) :=
begin
  rintros ⟨p_f, hf, k_f, hf_bound⟩ ⟨p_g, hg, k_g, hg_bound⟩,
  use (λ n, p_f (p_g n)),
  split,
  { intro n,
    apply le_trans (hf (g n)),
    apply hf },
  { use (k_f * k_g),
    intro n,
    apply le_trans (hf_bound (p_g n)),
    apply pow_mul_le_pow,
    apply hg_bound }
end

-- Polynomial composition is a polynomial
example (a b c : ℕ) :
  is_polynomial (λ x, a * x ^ 2 + b * x + c) :=
begin
  use (λ x, a * x ^ 2 + b * x + c),
  split,
  { intro n, refl },
  { use 2, intro n, simp, ring }
end

-- Verify polynomial composition
example (a b c d e f : ℕ) :
  is_polynomial ((λ x, a * x ^ 2 + b * x + c) ∘ (λ x, d * x ^ 2 + e * x + f)) :=
begin
  apply composition_polynomial,
  { use (λ x, a * x ^ 2 + b * x + c),
    split,
    { intro n, refl },
    { use 2, intro n, simp, ring } },
  { use (λ x, d * x ^ 2 + e * x + f),
    split,
    { intro n, refl },
    { use 2, intro n, simp, ring } }
end
```
