THEOREM: The composition of two polynomial-time computable functions is also polynomial-time computable.

PROOF SKETCH:
1. **Definitions**: Let f and g be polynomial-time computable functions. By definition, there exist polynomials p and q such that the time complexity of f is bounded by p and the time complexity of g is bounded by q.
2. **Composition**: Consider the composition function h(x) = f(g(x)). 
3. **Time Complexity**: The time to compute h(x) is the time to compute g(x) plus the time to compute f(g(x)). 
4. **Polynomial Bound**: Since the size of g(x) is polynomially bounded by the size of x (because g is polynomial-time), the time to compute f(g(x)) is bounded by p(q(|x|)), which is still a polynomial in |x|.
5. **Conclusion**: Therefore, h is polynomial-time computable.

THEOREM: The composition of a polynomial-time computable function with another polynomial-time computable function is itself polynomial-time computable.

PROOF SKETCH: 
1. **Definitions**: Recall that a function f: Σ* → Σ* is polynomial-time computable if there exists a Turing machine M that computes f and runs in time p(|x|) for some polynomial p and input x.
2. **Composition**: Given two polynomial-time computable functions f and g, we need to show that h(x) = f(g(x)) is also polynomial-time computable.
3. **Turing Machine Construction**: Construct a Turing machine M_h that simulates M_g on input x to compute g(x), then simulates M_f on the result to compute f(g(x)).
4. **Runtime Analysis**: The total running time of M_h is the sum of the running times of M_g and M_f. Since both are polynomial in |x|, their sum is also polynomial in |x| (as the sum of two polynomials is a polynomial).
5. **Conclusion**: Therefore, h is polynomial-time computable.

LEAN HINT:
- Imports: `import complexity.basic`
- Tactics: `apply`, `use` (for existential instantiation), `simp` (for polynomial arithmetic), `exact` (to supply the composed polynomial)
