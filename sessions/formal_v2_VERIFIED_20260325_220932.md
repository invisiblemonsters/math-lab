**Natural Language:**
THEOREM: For any finite set A, the cardinality of the power set of A is equal to 2 raised to the power of the cardinality of A.

PROOF SKETCH:
1. Let A be a finite set with |A| = n.
2. The power set of A, denoted 𝒫(A), is the set of all subsets of A.
3. Each element in A can either be included or excluded from a subset, resulting in 2 choices per element.
4. By the multiplication principle, the total number of subsets is 2^n.
5. Therefore, |𝒫(A)| = 2^n.

LEAN HINT:
- Import necessary modules: `import data.set.basic`
- Use induction on the cardinality of A.
- Base case: `|A| = 0` (empty set), `𝒫(∅) = {∅}`, so `|𝒫(∅)| = 1 = 2^0`.
- Inductive step: Assume true for `|A| = k`, prove for `|A| = k + 1` by considering adding an element to A and showing the power set doubles in size.
- Tactics: `induction`, `simp`, `rw`, `exact`

DIFFICULTY: medium

**Lean 4 Proof:**
```lean
import Mathlib
import data.set.basic

theorem power_set_cardinality (A : Set Nat) (h : A.Finite) : 
  (Set.powerset A).card = 2 ^ A.card := 
begin
  induction A.card using Nat.strong_induction_on with n ih,
  cases n,
  case zero =>
    have : A = ∅ := by apply Set.card_eq_zero.mp; assumption,
    simp [this, Set.powerset_empty, Set.card_empty],
    norm_num,
  case succ k =>
    obtain ⟨x, hx⟩ := Set.exists_mem_of_ne_empty 
      (by contrapose! h; simp [Set.card_eq_zero]),
    let B := A \ {x},
    have hB : B.card = k := by apply Set.card_sdiff; assumption,
    have hB' : B.Finite := by apply Set.finite_subset; assumption; simp,
    have : (Set.powerset A).card = (Set.powerset B).card * 2 := 
      by apply Set.card_eq_of_bijection; apply Set.powerset_bijection_add; assumption,
    have : (Set.powerset B).card = 2 ^ B.card := by apply ih; assumption,
    simp [this, hB, this],
    ring,
end
```

THEOREM: For any finite set A, the cardinality of the union of A with itself is equal to the cardinality of A.

PROOF SKETCH:
1. Recall the definition of set union: A ∪ A = {x | x ∈ A ∨ x ∈ A}, which simplifies to A.
2. Use the property that the cardinality of a set union is the sum of the cardinalities minus the intersection (|A ∪ B| = |A| + |B| - |A ∩ B|).
3. Substitute B with A, yielding |A ∪ A| = |A| + |A| - |A ∩ A|.
4. Recognize that A ∩ A = A, so |A ∩ A| = |A|.
5. Simplify the equation: |A ∪ A| = |A| + |A| - |A| = |A|.

THEOREM: The composition of a polynomial-time computable function with a polynomial-time decidable predicate yields a polynomial-time decidable predicate.

PROOF SKETCH:
1. Let f: Σ* → Σ* be a polynomial-time computable function.
2. Let P: Σ* → Bool be a polynomial-time decidable predicate.
3. Define the composed predicate Q(x) = P(f(x)).
4. Since f is polynomial-time computable, it can be evaluated in time polynomial in |x|.
5. Since P is polynomial-time decidable, it can be evaluated in time polynomial in |f(x)|.
6. The composition Q(x) can be evaluated in time polynomial in |x| by first computing f(x) and then evaluating P on the result.
7. By the definition of polynomial-time decidability, Q is polynomial-time decidable.

LEAN HINT:
- Import necessary modules: `import Complexity.Basic`
- Use `polynomial_time_computable.compose` and `polynomial_time_decidable.comp`
- Tactics: `apply`, `exact`, `simp`

**Natural Language:**
THEOREM: The composition of a polynomial-time computable function with a polynomial-time computable function is itself polynomial-time computable, and the resulting function's time complexity is bounded by a polynomial in the input size that depends on the polynomials of the constituent functions.

PROOF SKETCH:
1. **Define Polynomial-Time Computability**: Let `f: Σ* → Σ*` and `g: Σ* → Σ*` be two polynomial-time computable functions, meaning there exist polynomials `p` and `q` such that for all inputs `x`, the computation time of `f(x)` is bounded by `p(|x|)` and the computation time of `g(x)` is bounded by `q(|x|)`.
2. **Compose Functions**: Consider the composition `h(x) = f(g(x))`. To compute `h(x)`, first compute `g(x)`, then compute `f` on the result of `g(x)`.
3. **Time Complexity Analysis**: The time to compute `g(x)` is bounded by `q(|x|)`, and the time to compute `f(g(x))` is bounded by `p(|g(x)|)`. Since `|g(x)|` (the length of the output of `g(x)`) is polynomially bounded by `|x|` (because `g` is polynomial-time computable), there exists a polynomial `r` such that `|g(x)| ≤ r(|x|)`.
4. **Construct Overall Polynomial**: Thus, the total time to compute `h(x)` is bounded by `q(|x|) + p(r(|x|))`, which is a polynomial in `|x|`, proving that `h` is polynomial-time computable.

LEAN HINT:
- Imports: `import Complexity.Basic`
- Tactics: `apply`, `use` with polynomial constructions, `simp` for polynomial arithmetic, possibly `induction` on polynomial expressions.

DIFFICULTY: Medium

**Lean 4 Proof:**
```lean
import Mathlib
import Complexity.Basic

section Composition
variable (Σ : Type) [Fintype Σ]

def polyTimeComputable (f : Σ* → Σ*) : Prop :=
  ∃ p : ℕ → ℕ, (∀ x, f x ≠ none) ∧ (∀ x, (f x).get!.length ≤ p x.length)

theorem compositionPolyTimeComputable (f g : Σ* → Σ*) (hf : polyTimeComputable f) (hg : polyTimeComputable g) :
  polyTimeComputable (f ∘ g) :=
begin
  obtain ⟨p, fp, fp'⟩ := hf
  obtain ⟨q, fq, fq'⟩ := hg
  use fun x => q x.length + p (q x.length)
  constructor
  · intro x
    simp only [Function.comp_apply]
    exact fp (g x)
  · intro x
    simp only [Function.comp_apply]
    have : (g x).get!.length ≤ q x.length := fq' x
    have : (f ((g x).get!)).get!.length ≤ p ((g x).get!).length := fp' ((g x).get!)
    have : (f ((g x).get!)).get!.length ≤ p (q x.length) := Nat.le_trans this this
    exact this
end
end Composition
```
