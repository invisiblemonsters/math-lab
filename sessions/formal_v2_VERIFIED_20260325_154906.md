THEOREM: If a language L is in NP, then its complement (the set of all strings not in L) is in co-NP. Formally, L ∈ NP implies ¬L ∈ co-NP.

PROOF SKETCH: 
1. Recall the definition of NP: a language L is in NP if there exists a polynomial-time decidable relation R and a polynomial p such that for all x, x ∈ L iff there exists a certificate y with |y| ≤ p(|x|) and R(x, y) holds.
2. The complement of L, denoted ¬L, consists of all strings x not in L. By definition of co-NP, ¬L is in co-NP if there exists a polynomial-time decidable relation S and a polynomial q such that for all x, x ∈ ¬L iff for all certificates z with |z| ≤ q(|x|), S(x, z) does not hold.
3. Construct S from R: define S(x, z) as the negation of R(x, z), i.e., S(x, z) = ¬R(x, z). Since R is polynomial-time decidable, so is S (using the closure of decidable relations under negation, which follows from previous results on decidable languages).
4. Show that x ∈ ¬L iff for all z with |z| ≤ p(|x|), S(x, z) holds. This follows directly from the definition of ¬L and the properties of R.
5. Conclude that ¬L ∈ co-NP by verifying the co-NP conditions with S and p.

LEAN HINT:
```lean
import computability.complexity.basic

theorem np_complement_in_conp (L : language) (h : L ∈ NP) : ¬L ∈ coNP :=
begin
  -- Unfold the definitions of NP and co-NP
  cases h with R p hR hp,
  -- Construct the relation S as the negation of R
  let S := λ x z, ¬R x z,
  -- Show S is polynomial-time decidable using closure under negation
  have hS : polynomial_time_decidable S := 
    by apply decidable_negationclosure; exact hR,
  -- Verify the co-NP condition for ¬L using S and p
  use [S, p],
  intros x,
  exact iff.intro 
    (λ hxl, ∀z (hz : |z| ≤ p(|x|)), S x z) 
    (λ hxl, ¬∃z (hz : |z| ≤ p(|x|)), R x z),
  -- Apply iff equivalence and simplify using definitions
  simp [*, hp], 
  apply exists_conp_iff_not_forall,
end
```

THEOREM: The complexity class P is closed under the intersection operation. Specifically, if languages L₁ and L₂ over an alphabet Σ are in P, then their intersection L₁ ∩ L₂ is also in P.

PROOF SKETCH: 
1. **Definition Recall**: Start by recalling the definition of P as the set of languages decidable by a deterministic Turing machine in polynomial time.
2. **Intersection as Decision Problem**: Recognize that the intersection L₁ ∩ L₂ corresponds to the decision problem where a string is accepted if and only if it is accepted by both L₁ and L₂.
3. **Constructing the TM for Intersection**: Describe the construction of a deterministic Turing machine M for L₁ ∩ L₂ that simulates the Turing machines for L₁ and L₂ sequentially. 
4. **Polynomial Time Analysis**: Argue that the total running time of M is the sum of the running times of the simulations for L₁ and L₂. Since both are polynomial in the input length, their sum is also polynomial.
5. **Conclusion**: Conclude that since M decides L₁ ∩ L₂ in polynomial time, L₁ ∩ L₂ is in P.

LEAN HINT:

```lean
theorem P_closed_under_intersection 
  (L₁ L₂ : language) (h₁ : L₁ ∈ P) (h₂ : L₂ ∈ P) :

  -- Unfold definitions of P and language membership
  unfold mem_P at *,
  -- Construct the simulator for L₁ ∩ L₂ using h₁ and h₂

  -- Verify the simulator's correctness

  -- Establish polynomial time bound

  have h₂_poly := h₂.poly_time,
  -- Use polynomial addition closure (from previous phases)

  -- Simplify to match the required form
```

THEOREM: The complexity class NP is closed under the union operation. Specifically, if languages L₁ and L₂ over an alphabet Σ are in NP, then their union L₁ ∪ L₂ is also in NP.

PROOF SKETCH: 
1. **Definition of NP**: Recall that a language L is in NP if there exists a polynomial p and a relation R such that:
   - For all x, x ∈ L iff there exists a certificate y with |y| ≤ p(|x|) and R(x, y) holds.
2. **Constructing the Verifier for L₁ ∪ L₂**:
   - For the union, we need to show that there exists a polynomial-time verifiable certificate for x being in either L₁ or L₂.
   - Given x, the certificate can be a pair (b, y), where b is a bit indicating whether the certificate is for L₁ (b=0) or L₂ (b=1), and y is the certificate for the corresponding language.
   - The verifier checks if b=0 and y is a valid certificate for L₁, or b=1 and y is a valid certificate for L₂. This can be done in polynomial time by running the verifiers for L₁ and L₂ in parallel.
3. **Polynomial Bound**: The size of the certificate (b, y) is bounded by the maximum of the polynomial bounds for L₁ and L₂ plus a constant (for the bit b), which is still a polynomial.
