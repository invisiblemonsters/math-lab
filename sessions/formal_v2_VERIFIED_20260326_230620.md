THEOREM: The sum of two polynomial functions over the natural numbers is also a polynomial function. Formally, if f, g : ℕ → ℕ are polynomial functions, then the function h : ℕ → ℕ defined by h(n) = f(n) + g(n) is also a polynomial function.

PROOF SKETCH:
1. **Definition of Polynomial Function**: Recall that a function f : ℕ → ℕ is a polynomial function if there exists a polynomial p(x) with non-negative integer coefficients such that f(n) = p(n) for all n ∈ ℕ.
2. **Sum of Polynomials**: Use the fact that the sum of two polynomials is also a polynomial. Specifically, if f(n) = p(n) and g(n) = q(n), then h(n) = p(n) + q(n), which is a polynomial with coefficients obtained by adding the corresponding coefficients of p and q.
3. **Non-Negativity of Coefficients**: Show that the coefficients of the resulting polynomial are non-negative, as the sum of non-negative coefficients is also non-negative.
4. **Conclusion**: Conclude that h is a polynomial function by definition.

LEAN HINT:
- Imports: `import algebra.polynomial`
- Tactics: `simp`, `ring`, `exact`, `apply`, `intro`
- Consider defining polynomial functions using `polynomial.ℕ[X]` and using the `eval` function to evaluate polynomials at natural numbers.

import algebra.polynomial

def is_polynomial_function (f : ℕ → ℕ) : Prop :=
∃ p : polynomial ℕ, ∀ n : ℕ, f n = polynomial.eval n p

theorem sum_of_polynomial_functions (f g : ℕ → ℕ) :
is_polynomial_function f → is_polynomial_function g → is_polynomial_function (λ n, f n + g n) :=
begin
  intro hf hg,
  cases hf with pf hf,
  cases hg with pg hg,
  use pf + pg,
  intro n,
  simp [polynomial.eval_add],
  rw [hf n, hg n],
  ring,
end

theorem non_negativity_of_coefficients (p q : polynomial ℕ) :
(∀ i, 0 ≤ polynomial.coeff p i) → (∀ i, 0 ≤ polynomial.coeff q i) → (∀ i, 0 ≤ polynomial.coeff (p + q) i) :=
begin
  intro hp hq i,
  simp [polynomial.coeff_add],
  apply add_nonneg,
  exact hp i,
  exact hq i,
end

theorem sum_of_polynomial_functions_is_polynomial :
∀ f g : ℕ → ℕ, is_polynomial_function f → is_polynomial_function g →
∃ p : polynomial ℕ, (∀ i, 0 ≤ polynomial.coeff p i) ∧ ∀ n, f n + g n = polynomial.eval n p :=
begin
  intro f g hf hg,
  cases hf with pf hf,
  cases hg with pg hg,
  use pf + pg,
  split,
  apply non_negativity_of_coefficients,
  intro i,
  cases (classical.em (polynomial.coeff pf i = 0)) with hcoeff hcoeff,
  simp [hcoeff],
  apply add_nonneg,
  apply zero_le,
  apply zero_le,
  intro i,
  apply zero_le,
  intro n,
  simp [polynomial.eval_add],
  rw [hf n, hg n],
  ring,
end

PROOF SKETCH:

2. **Composition Time Analysis**: Consider the composition h(x) = f(g(x)). The time to compute g(x) is bounded by some polynomial q(|x|), and the time to compute f(g(x)) is bounded by some polynomial r(|g(x)|).
3. **Polynomial Bound**: Since |g(x)| is bounded by q(|x|) (as g is polynomial-time), we can substitute this into the bound for f, yielding a polynomial in |x|, specifically r(q(|x|)), which is still a polynomial.
4. **Conclusion**: Therefore, the total time to compute h(x) is bounded by a polynomial in |x|, proving that h is polynomial-time.

**Natural Language:**
THEOREM: The set of decidable languages is closed under the union operation. Formally, if L₁ and L₂ are decidable languages, then the language L₁ ∪ L₂ = {x | x ∈ L₁ ∨ x ∈ L₂} is also decidable.

PROOF SKETCH: 
1. **Definition Recall**: Start by recalling the definition of a decidable language, which is a language for which there exists a Turing machine (or an algorithm) that halts and accepts or rejects any input string.
2. **Union Construction**: To show that L₁ ∪ L₂ is decidable, construct a Turing machine that, on input x, simulates the decision procedures for L₁ and L₂ in parallel.
3. **Simulation Argument**: Argue that this simulation can be done by interleaving the steps of the two machines. If either machine halts and accepts, the constructed machine halts and accepts. If both machines halt and reject, the constructed machine halts and rejects.
4. **Halting Guarantee**: Since L₁ and L₂ are decidable, both simulations are guaranteed to halt. Thus, the constructed machine for L₁ ∪ L₂ also halts on all inputs, ensuring decidability.

LEAN HINT:
- Imports: `import computability.turing_machine`
- Tactics: `apply`, `use` (to construct the Turing machine), `simp` (to verify the halting condition), `exact` (to reference decidability of L₁ and L₂)

DIFFICULTY: medium

**Lean 4 Proof:**
```lean
import Mathlib
import computability.turing_machine

-- Define decidability of a language
def decidable_language (L : set string) : Prop :=
  ∃ (M : turing_machine), ∀ (x : string), (x ∈ L → M.accepts x) ∧ (x ∉ L → M.rejects x)

-- Theorem statement
theorem decidable_union (L₁ L₂ : set string) (h₁ : decidable_language L₁) (h₂ : decidable_language L₂) :
  decidable_language (L₁ ∪ L₂) :=
begin
  -- Extract the Turing machines for L₁ and L₂
  obtain ⟨M₁, h₁⟩ := h₁
  obtain ⟨M₂, h₂⟩ := h₂

  -- Construct the Turing machine for L₁ ∪ L₂
  use (interleave M₁ M₂)

  -- Verify the halting condition
  intro x
  simp only [interleave_accepts_iff, interleave_rejects_iff]
  cases (x ∈ L₁ ∨ x ∈ L₂)

  -- Case 1: x ∈ L₁ ∨ x ∈ L₂
  case inl hx :
    cases hx
    case inl hx₁ :
      exact Or.inl (h₁.1 x hx₁)
    case inr hx₂ :
      exact Or.inr (h₂.1 x hx₂)

  -- Case 2: x ∉ L₁ ∪ L₂
  case inr hx :
    have hx₁ : x ∉ L₁ := by simp [hx]
    have hx₂ : x ∉ L₂ := by simp [hx]
    exact And.intro (h₁.2 x hx₁) (h₂.2 x hx₂)
end
```
