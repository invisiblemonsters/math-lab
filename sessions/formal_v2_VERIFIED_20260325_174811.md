THEOREM: For any natural numbers a, b, and c, if a ≤ b, then a + c ≤ b + c.

PROOF SKETCH:
1. **Assume** a ≤ b (i.e., there exists a natural number d such that a + d = b).
2. **Add c to both sides** of the equation a + d = b, resulting in a + c + d = b + c.
3. **Apply the definition of ≤** for natural numbers: a + c ≤ b + c because there exists a natural number d such that (a + c) + d = b + c.

LEAN HINT:

  -- Use the exists.intro to extract the witness d

  -- The rest should be straightforward with simp
  simp,
end

THEOREM: For all natural numbers n and m, if n ≤ m, then the length of the list [n, n+1, ..., m] is equal to m - n + 1.

PROOF SKETCH:
1. **Base Case**: If n = m, the list is [n], and its length is 1, which equals m - n + 1 = 1.
2. **Inductive Step**: Assume the statement holds for some n and m where n ≤ m. Consider n and m+1. If n ≤ m+1, then either n = m+1 (trivial) or n ≤ m. In the latter, the list [n, ..., m, m+1] has length (m - n + 1) + 1 = (m+1) - n + 1, which matches the formula.

PROOF SKETCH:
1. **Decidability of L**: Since L is polynomial-time decidable, there exists a polynomial p and a Turing machine M that decides L in time p(|w|) for input w.
2. **Polynomial-time composition**: The composition of f and g, denoted g ∘ f, is polynomial-time by previous theorems (specifically, theorem 2).
3. **Constructing the decider for { x | g(f(x)) ∈ L }**:
   - Given input x, compute f(x) in polynomial time.
   - Then compute g(f(x)) in polynomial time.
   - Use the decider for L to check if g(f(x)) ∈ L, which also operates in polynomial time.
4. **Overall polynomial time**: The total time is the sum of the times for f, g, and the decider for L, each of which is polynomial in |x|. Since the sum of polynomials is a polynomial, the constructed decider runs in polynomial time.

THEOREM: The composition of a polynomial function with another polynomial function results in a polynomial function. Specifically, if f: ℕ → ℕ and g: ℕ → ℕ are polynomially bounded functions, then their composition f ∘ g is also polynomially bounded.

PROOF SKETCH:
1. **Polynomial Bounds**: Assume f(n) ≤ a * n^k and g(n) ≤ b * n^m for constants a, b, k, m ∈ ℕ.
2. **Substitute g into f**: Compute f(g(n)) ≤ a * (g(n))^k ≤ a * (b * n^m)^k.
3. **Simplify**: This simplifies to a * b^k * n^{m*k}, which is a polynomial in n.
4. **Conclusion**: Thus, f ∘ g is bounded by a polynomial, specifically of degree m*k.

**Natural Language:**
THEOREM: The set of polynomial-time decidable languages over an alphabet Σ is closed under the union operation. Specifically, if languages L₁ and L₂ over Σ are polynomial-time decidable, then their union L₁ ∪ L₂ is also polynomial-time decidable.

PROOF SKETCH:
1. **Decidability of L₁ and L₂**: Assume we have polynomial-time deciders for L₁ and L₂, i.e., Turing machines M₁ and M₂ that decide L₁ and L₂, respectively, in polynomial time.
2. **Constructing the Decider for L₁ ∪ L₂**: Design a Turing machine M that, on input a string w, simulates both M₁ and M₂ on w in parallel (or sequentially, but within polynomial time bounds).
3. **Polynomial Time Bound**: Show that the running time of M is bounded by a polynomial in the length of w. Since both M₁ and M₂ run in polynomial time, the maximum running time of the two (plus a constant overhead for the simulation) is still polynomial.
4. **Correctness**: Argue that M correctly decides the union language L₁ ∪ L₂ by accepting w if either M₁ or M₂ accepts w.

LEAN HINT:
- Imports: `import computability.turing_machine`, `import data.polynomial`
- Tactics: `apply`, `use` (to construct the simulator), `simp` (for polynomial bounds), `linarith` (for verifying polynomial inequalities)

DIFFICULTY: medium

**Lean 4 Proof:**
```lean
import Mathlib
import computability.turing_machine
import data.polynomial

theorem union_decidable {Σ : Type} [fintype Σ] (L₁ L₂ : set (list Σ))
  (d₁ : turing_decidable L₁) (d₂ : turing_decidable L₂) :
  turing_decidable (L₁ ∪ L₂) :=
begin
  rcases d₁ with ⟨M₁, h₁⟩,
  rcases d₂ with ⟨M₂, h₂⟩,
  apply turing_decidable.mk,
  use (turing_machine.parallel_or M₁ M₂),
  intro w,
  simp only [turing_machine.parallel_or_accepts_iff],
  split,
  { rintro (h | h),
    { exact or.inl (h₁ _ h) },
    { exact or.inr (h₂ _ h) } },
  { rintro (h | h),
    { exact or.inl (h₁ _ h) },
    { exact or.inr (h₂ _ h) } },
  intro n,
  let m₁ := turing_machine.steps M₁ w n,
  let m₂ := turing_machine.steps M₂ w n,
  let m := max m₁ m₂,
  have hm : m ≤ n ^ 2 := by linarith [h₁ _ n, h₂ _ n],
  use m,
  simp only [turing_machine.parallel_or_steps],
  linarith [hm]
end
```
