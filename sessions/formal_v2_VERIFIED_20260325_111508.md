THEOREM: The composition of two polynomial-time functions is itself a polynomial-time function. Specifically, if \( f(n) \) is a polynomial-time function and \( g(n) \) is a polynomial-time function, then \( h(n) = f(g(n)) \) is also a polynomial-time function.

PROOF SKETCH:
1. **Definitions**: Recall that a polynomial-time function is one that can be computed in time bounded by a polynomial in the input size.
2. **Composition**: Consider the composition \( h(n) = f(g(n)) \). Since \( g(n) \) is polynomial-time, it runs in time \( p(n) \) for some polynomial \( p \).
3. **Substitution**: The output of \( g(n) \) is of size at most \( p(n) \), which serves as the input to \( f \). Since \( f \) is polynomial-time, it runs in time \( q(p(n)) \) for some polynomial \( q \).
4. **Polynomial Bound**: The total time for \( h(n) \) is \( p(n) + q(p(n)) \), which is still a polynomial in \( n \) (specifically, \( p(n) + q(p(n)) \leq r(n) \) for some polynomial \( r \)).
5. **Conclusion**: Therefore, \( h(n) \) is polynomial-time.

LEAN HINT:
- Imports: `import Data.Polynomial` and relevant complexity theory definitions
- Tactics: `simp` for polynomial arithmetic, `apply` with known polynomial bounds, `intro` for function composition

import Data.Polynomial

-- Define a polynomial-time function
def PolyTime (f : ℕ → ℕ) : Prop := ∃ p : ℕ[X], ∀ n, f n ≤ p.eval n

-- The composition of two polynomial-time functions is itself a polynomial-time function
theorem composition_polytime (f g : ℕ → ℕ) (hf : PolyTime f) (hg : PolyTime g) : PolyTime (f ∘ g) := 
begin
  -- Extract the polynomials bounding f and g
  obtain ⟨p, hp⟩ := hf,
  obtain ⟨q, hq⟩ := hg,
  -- Define the polynomial bounding the composition
  let r : ℕ[X] := p.comp q,
  -- Show that r bounds the composition
  apply Exists.intro r,
  intro n,
  -- Bound the composition using the triangle inequality
  have h1 : (f ∘ g) n = f (g n) := rfl,
  have h2 : f (g n) ≤ p.eval (g n) := hp (g n),
  have h3 : g n ≤ q.eval n := hq n,
  have h4 : p.eval (g n) ≤ p.eval (q.eval n) := Nat.monotone_eval h3,
  -- Combine the inequalities to get the final bound

   - If any decomposition satisfies the condition, accept w; otherwise, reject w.
4. **Termination Argument**:
   - The iteration over decompositions is guaranteed to terminate because the length of w is finite, and the number of possible decompositions, though exponential, is finite and systematically enumerable.

5. **Correctness Proof**:
   - **Soundness**: If the algorithm accepts w, then by construction, w is a concatenation of strings in L, hence w ∈ L*.

   - **Correctness**: By the reduction property, x ∈ L₂ ⇨ f(x) ∈ L₁ ⇨ D accepts f(x) ⇨ D' accepts x. Similarly for rejection.
   - **Termination**: f is polynomial-time computable (hence always halts), and D always halts. Thus, D' always halts.
5. **Conclude** L₂ is decidable.

THEOREM: The complexity class P is closed under the symmetric difference operation. Specifically, if L₁ and L₂ are languages in P, then the symmetric difference of L₁ and L₂ (denoted L₁ Δ L₂) is also in P.

PROOF SKETCH: 
1. Recall the definition of symmetric difference: L₁ Δ L₂ = (L₁ \ L₂) ∪ (L₂ \ L₁).
2. Since P is closed under union (from previously verified theorems, e.g., THEOREM 12) and set difference (can be derived from closure under intersection and complement, noting that set difference is L₁ \ L₂ = L₁ ∩ L₂ᶜ), we can use these properties.
3. Given L₁ and L₂ are in P, their complements L₁ᶜ and L₂ᶜ are also in P (since P is closed under complement, as decidability in polynomial time implies the complement is also decidable in polynomial time).
4. Therefore, L₁ ∩ L₂ᶜ and L₂ ∩ L₁ᶜ are in P (using closure under intersection).
5. The union of these two intersections, which is L₁ Δ L₂, is also in P (using closure under union).

LEAN HINT:
- Imports: `import complexity.basic`
- Tactics: `apply`, `exact`, `use` closure properties of P (union, intersection, complement), potentially using `simp` for set operations.

import Mathlib

namespace Language

def symmetric_difference (L₁ L₂ : Language) : Language := (L₁ \ L₂) ∪ (L₂ \ L₁)

theorem symmetric_difference_in_P (L₁ L₂ : Language) 
  (h₁ : L₁ ∈ P) 
  (h₂ : L₂ ∈ P) 
  (P_closed_under_complement : ∀ L, L ∈ P → Lᶜ ∈ P) 
  (P_closed_under_intersection : ∀ L₁ L₂, L₁ ∈ P → L₂ ∈ P → L₁ ∩ L₂ ∈ P) 
  (P_closed_under_union : ∀ L₁ L₂, L₁ ∈ P → L₂ ∈ P → L₁ ∪ L₂ ∈ P) 
  : L₁ Δ L₂ ∈ P := 
begin
  have h_sym_diff : L₁ Δ L₂ = (L₁ \ L₂) ∪ (L₂ \ L₁) := rfl,
  have h_L₁_minus_L₂ : L₁ \ L₂ = L₁ ∩ L₂ᶜ := rfl,
  have h_L₂_minus_L₁ : L₂ \ L₁ = L₂ ∩ L₁ᶜ := rfl,
  have h_L₁_complement : L₁ᶜ ∈ P := P_closed_under_complement L₁ h₁,
  have h_L₂_complement : L₂ᶜ ∈ P := P_closed_under_complement L₂ h₂,
  have h_L₁_cap_L₂_complement : L₁ ∩ L₂ᶜ ∈ P := P_closed_under_intersection L₁ L₂ᶜ h₁ h_L₂_complement,
  have h_L₂_cap_L₁_complement : L₂ ∩ L₁ᶜ ∈ P := P_closed_under_intersection L₂ L₁ᶜ h₂ h_L₁_complement,
  rw [h_sym_diff, h_L₁_minus_L₂, h_L₂_minus_L₁],
  apply P_closed_under_union (L₁ ∩ L₂ᶜ) (L₂ ∩ L₁ᶜ),
  exact h_L₁_cap_L₂_complement,
  exact h_L₂_cap_L₁_complement,
end

end Language

THEOREM: If a language L is in P, then its complement (the set of all strings not in L) is also in P.

PROOF SKETCH: 
1. Recall the definition of P: a language L is in P if there exists a polynomial-time decidable algorithm for L.
2. To show the complement of L (denoted ¬L) is in P, we need to construct a polynomial-time decidable algorithm for ¬L.
3. Given a string w, the algorithm for ¬L can simply run the polynomial-time algorithm for L on w and return the opposite result.
4. Since the original algorithm for L runs in polynomial time, the "opposite" algorithm for ¬L also runs in polynomial time (as it only adds a constant-time operation).
5. Therefore, ¬L is in P.

LEAN HINT:

import complexity.basic

theorem complement_in_P {L : language} (h : L ∈ P) : L.complement ∈ P :=
begin
  -- Recall the definition of P and complement
  unfold P at h,
  unfold language.complement,
  -- Construct the polynomial-time algorithm for ¬L
  apply exists.intro (λ w, decide_not (h.decider w)),
  -- Show the new algorithm runs in polynomial time
  apply polynomial_time.compose,
  -- Use the polynomial time property of L's algorithm
  exact h.poly_time,
  -- The "decide_not" operation is constant-time
  apply polynomial_time.const,
end

**Natural Language:**
THEOREM: The complexity class P is closed under the concatenation operation. Specifically, if L₁ and L₂ are languages in P, then the concatenation of L₁ and L₂ (denoted L₁ ◦ L₂) is also in P.

PROOF SKETCH:
1. Recall the definition of P: a language L is in P if there exists a deterministic Turing machine that decides L in polynomial time.
2. Assume L₁ and L₂ are in P. Thus, there exist polynomial-time decidable Turing machines M₁ and M₂ for L₁ and L₂, respectively.
3. Construct a new Turing machine M that decides L₁ ◦ L₂ by simulating M₁ on the prefix of the input string and M₂ on the suffix.
4. Show that the running time of M is polynomial in the length of the input string by analyzing the composition of the polynomial running times of M₁ and M₂.
5. Conclude that L₁ ◦ L₂ is in P by the definition of P.

LEAN HINT:
- Import necessary modules: `import computability.turing_machine`
- Define polynomial-time Turing machines for L₁ and L₂.
- Use `polynomial_time_compose` (from previous theorems) to establish the polynomial running time of the concatenated machine.
- Apply `decidable_concat` (to be proven) to show decidability of L₁ ◦ L₂.
- Tactics: `apply`, `exact`, `simp`, `omega` for polynomial time analysis.

DIFFICULTY: medium

```lean
import Mathlib
import computability.turing_machine

-- Define the complexity class P
def P (L : set string) : Prop := 
  ∃ (M : turing_machine), polynomial_time M ∧ (∀ x, M.accepts x = x ∈ L)

-- Assume L₁ and L₂ are in P
theorem concat_in_P (L₁ L₂ : set string) (h₁ : P L₁) (h₂ : P L₂) : P (L₁ ◦ L₂) := 
  begin
    -- Extract polynomial-time Turing machines for L₁ and L₂
    obtain ⟨M₁, hM₁, h₁⟩ := h₁,
    obtain ⟨M₂, hM₂, h₂⟩ := h₂,

    -- Construct a new Turing machine M that decides L₁ ◦ L₂
    let M := M₁.concat M₂,

    -- Show that the running time of M is polynomial in the length of the input string
    have hM : polynomial_time M := 
      begin
        apply polynomial_time_compose,
        exact hM₁,
        exact hM₂,
      end,

    -- Show that M decides L₁ ◦ L₂
    have h : ∀ x, M.accepts x = x ∈ (L₁ ◦ L₂) := 
      begin
        intro x,
        apply decidable_concat,
        exact h₁,
        exact h₂,
      end,

    -- Conclude that L₁ ◦ L₂ is in P
    exact ⟨M, hM, h⟩,
  end

-- Define the decidability of the concatenation of two languages
lemma decidable_concat (L₁ L₂ : set string) (h₁ : ∀ x, decidable (x ∈ L₁)) (h₂ : ∀ x, decidable (x ∈ L₂)) : 
  ∀ x, decidable (x ∈ (L₁ ◦ L₂)) := 
  begin
    intro x,
    cases h₁ (x.take (x.length / 2)) with h₁ h₁,
    cases h₂ (x.drop (x.length / 2)) with h₂ h₂,
    apply decidable.is_true,
    simp [set.mem_concat],
    exact ⟨h₁, h₂⟩,
    apply decidable.is_false,
    simp [set.mem_concat],
    intro h,
    cases h with h₁ h₂,
    contradiction,
    contradiction,
  end

-- Define the polynomial-time composition of two Turing machines
lemma polynomial_time_compose (M₁ M₂ : turing_machine) (h₁ : polynomial_time M₁) (h₂ : polynomial_time M₂) : 
  polynomial_time (M₁.concat M₂) := 
  begin
    simp [polynomial_time],
    intro x,
    cases h₁ x with n h₁,
    cases h₂ x with m h₂,
    apply exists.intro (n + m),
    simp [turing_machine.concat],
    omega,
  end
```
