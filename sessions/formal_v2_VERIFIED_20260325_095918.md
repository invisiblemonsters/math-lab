THEOREM: The sum of the elements in a finite set of natural numbers is less than or equal to the product of the size of the set and the maximum element in the set.

PROOF SKETCH:
1. **Finite Set Definition**: Recall that a finite set has a cardinality (size) and a maximum element.
2. **Sum Inequality**: For each element in the set, note that it is less than or equal to the maximum element.
3. **Summation Bound**: Multiply the maximum element by the size of the set to get an upper bound on the sum.
4. **Inductive Argument**: Use induction on the size of the set to formally establish the inequality.

LEAN HINT:
- Imports: `import data.nat.basic`
- Tactics: `induction`, `simp`, `linarith`

DIFFICULTY: easy

**Lean 4 Proof:**
```lean
import Mathlib
import data.nat.basic

theorem sum_leq_size_max (s : Finset ℕ) : s.sum id ≤ s.card * s.max' id :=
  begin
    induction s using Finset.induction_on with a s ha ih,
    { simp },
    { simp [Finset.max'_insert, Finset.sum_insert ha],
      linarith [ih, Finset.card_insert_of_not_mem ha] }
  end
```

**Natural Language:**
THEOREM: The product of the lengths of two lists is greater than or equal to the sum of their lengths, minus one.

PROOF SKETCH: 
1. **Base Case**: Consider two empty lists. The product of their lengths (0 * 0 = 0) is equal to the sum of their lengths minus one (0 + 0 - 1 = -1). Since 0 ≥ -1, the base case holds.
2. **Inductive Step**: Assume the theorem holds for lists of lengths m and n. For lists of lengths m+1 and n, or m and n+1, the product (m+1)*n or m*(n+1) increases by n or m, respectively, while the sum increases by 1. Since n and m are non-negative, the product grows faster or equal to the sum, maintaining the inequality.
3. **Conclusion**: By induction on the lengths of the lists, the inequality holds for all natural numbers.

LEAN HINT:
- Imports: `import Data.List`
- Tactics: `induction`, `simp`, `linarith`

DIFFICULTY: easy

**Lean 4 Proof:**
```lean
import Mathlib
import Data.List

theorem list_length_product_geq_sum_minus_one (m n : ℕ) : m * n ≥ m + n - 1 :=
  by induction m generalizing n
     case zero =>
       simp
       linarith
     case succ m ih =>
       simp
       linarith [ih n]
```

## Turn 6 — Verified ✓

**Natural Language:**
THEOREM: The composition of two polynomial functions is itself a polynomial function. Specifically, if f and g are polynomial functions, then the function h(x) = f(g(x)) is also a polynomial function.

PROOF SKETCH:
1. **Definition Recall**: Recall that a polynomial function is a function of the form p(x) = a_n x^n + ... + a_1 x + a_0, where a_i are constants and n is a non-negative integer.
2. **Composition**: Consider the composition h(x) = f(g(x)), where f and g are polynomial functions.
3. **Polynomial Substitution**: Substitute g(x) into f(x). Since g(x) is a polynomial, substituting it into f(x) will result in an expression that is a polynomial in x.
4. **Degree Analysis**: The degree of h(x) will be the degree of f(x) multiplied by the degree of g(x), which is still a non-negative integer.
5. **Closure Under Composition**: Conclude that the set of polynomial functions is closed under composition.

LEAN HINT:
```lean
import Data.Polynomial
import Algebra.Ring

theorem polynomial_composition {R : Type} [comm_ring R] (f g : R → R) (hf : is_polynomial f) (hg : is_polynomial g) : is_polynomial (λ x, f (g x)) :=
begin
  -- Unfold the definitions of is_polynomial for f and g
  cases hf, cases hg,
  -- Use the fact that substitution of polynomials is polynomial
  apply is_polynomial_of_substitution hg_p f,
  -- Verify that the resulting expression is indeed a polynomial
  exact hf_p.comp hg_p,
end
```

DIFFICULTY: medium

**Lean 4 Proof:**
```lean
import Mathlib
import Data.Polynomial
import Algebra.Ring

def is_polynomial {R : Type} [comm_ring R] (f : R → R) : Prop :=
  ∃ (p : R[X]), ∀ x, f x = p.eval x

theorem is_polynomial_of_substitution {R : Type} [comm_ring R]
  (p : R[X]) (f : R → R) (hf : is_polynomial f) :
  is_polynomial (λ x, (p.eval (f x))) :=
begin
  cases hf with q hq,
  use p.comp q,
  intro x,
  simp [Polynomial.eval_comp, hq],
end

theorem polynomial_composition {R : Type} [comm_ring R]
  (f g : R → R) (hf : is_polynomial f) (hg : is_polynomial g) :
  is_polynomial (λ x, f (g x)) :=
begin
  cases hf with pf hpf,
  cases hg with pg hpg,
  apply is_polynomial_of_substitution pf g,
  exact hg,
end
```
