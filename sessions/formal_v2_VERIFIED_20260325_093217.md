**Natural Language:**
THEOREM: The image of a finite set of natural numbers under a function is also finite.

PROOF SKETCH:
1. **Definition Recall**: A set is finite if it has a finite number of elements.
2. **Function Application**: Consider a function f: ℕ → ℕ and a finite set S = {a₁, a₂, ..., aₙ}.
3. **Image Construction**: The image f(S) = {f(a₁), f(a₂), ..., f(aₙ)}.
4. **Finiteness Preservation**: Since S has n elements, f(S) has at most n elements (due to potential duplicates), which is still finite.

LEAN HINT:
- Imports: `import data.set.finite`
- Tactics: `intro`, `apply`, `simp`, `exact` (using `set.finite.image` from Mathlib)

DIFFICULTY: easy

**Lean 4 Proof:**
```lean

import data.set.finite

theorem finite_image (f : ℕ → ℕ) (S : Set ℕ) (h : S.Finite) : (f '' S).Finite :=
  by exact Set.Finite.image h f
```

## Turn 5 — Verified ✓

**Natural Language:**
THEOREM: For any natural numbers a, b, and c, if a + b = c, then the maximum of a and b is less than or equal to c.

PROOF SKETCH:
1. Assume a + b = c.
2. Without loss of generality, suppose a ≤ b. Then, max(a, b) = b.
3. From a + b = c and a ≤ b, we have b ≤ c (since a is non-negative, adding it to b cannot decrease the sum).
4. Thus, max(a, b) = b ≤ c.
5. If b ≤ a, a similar argument shows max(a, b) = a ≤ c.
6. Therefore, in all cases, max(a, b) ≤ c.

LEAN HINT:
- Imports: `import data.nat.basic`
- Tactics: `intro`, `cases` (on the comparison of a and b), `linarith` or `omega` for inequality reasoning.

DIFFICULTY: easy

**Lean 4 Proof:**
```lean
import Mathlib

theorem max_le_sum (a b c : Nat) (h : a + b = c) : max a b ≤ c := 
  by cases le_or_lt a b 
    case inl h' => 
      have : max a b = b := max_eq_right h'
      rw [this]
      exact le_of_add_le_left h
    case inr h' => 
      have : max a b = a := max_eq_left (le_of_lt h')
      rw [this]
      exact le_of_add_le_right h
```

2. Let m = max(S), the maximum element in S.

THEOREM: The composition of a polynomial-time computable function with a polynomial-time computable relation yields a polynomial-time computable relation.

PROOF SKETCH:
1. **Definitions**: Recall the definitions of polynomial-time computable functions and relations. A relation R is polynomial-time computable if there exists a polynomial p such that deciding R(x, y) can be done in time p(|x| + |y|).
2. **Composition**: Consider a polynomial-time computable function f and a polynomial-time computable relation R. We need to show that the relation S defined by S(x, y) = R(x, f(y)) is also polynomial-time computable.
3. **Time Complexity Analysis**: The time to compute f(y) is bounded by a polynomial q(|y|). The time to decide R(x, f(y)) is bounded by a polynomial p(|x| + |f(y)|). Since |f(y)| is bounded by a polynomial in |y| (because f is polynomial-time computable), the total time is bounded by a polynomial in |x| + |y|.

THEOREM: The sum of two polynomial functions is itself a polynomial function. Specifically, if f and g are polynomial functions, then the function h defined by h(n) = f(n) + g(n) for all natural numbers n is also a polynomial function.

PROOF SKETCH: 
1. Recall the definition of a polynomial function, which is a function that can be expressed as a polynomial in n with non-negative integer coefficients (e.g., f(n) = a_k n^k + ... + a_1 n + a_0).
2. Assume f and g are polynomial functions of degrees m and p, respectively.
3. When we add f(n) and g(n), the resulting function h(n) will have a degree equal to the maximum of m and p.
4. The coefficients of h(n) will be the sum of the corresponding coefficients of f(n) and g(n).
5. Since the sum of polynomials is a polynomial, h(n) is also a polynomial function.

- Use the `polynomial` type and its operations (e.g., `add_poly`).

**Natural Language:**
THEOREM: The composition of two polynomial functions is itself a polynomial function. Specifically, if f and g are polynomial functions, then the function h defined by h(n) = f(g(n)) is also a polynomial function.

PROOF SKETCH: To prove this, we start by recalling the definition of a polynomial function, which is a function that can be expressed as a sum of terms of the form a_k n^k, where a_k are coefficients and k is a non-negative integer. Given two polynomial functions f and g, we can express f as a sum of terms with coefficients a_i and exponents i, and g as a sum of terms with coefficients b_j and exponents j. The composition h(n) = f(g(n)) involves substituting g(n) into f. By expanding this substitution, we can show that h(n) can also be expressed as a sum of terms with coefficients (which are products and sums of the original coefficients) and exponents that are products of the original exponents. Since the exponents in a polynomial function are bounded by the degree of the polynomial, and the coefficients are finite, the resulting function h(n) will also have a finite degree and thus be a polynomial function. This can be formalized using properties of polynomial arithmetic and induction on the degrees of f and g.

LEAN HINT: 
- Imports: `import algebra.polynomial`
- Tactics: `ring`, `induction` on polynomial degrees, `simp` for expanding polynomial expressions.

DIFFICULTY: medium

**Lean 4 Proof:**
```lean
import Mathlib
import algebra.polynomial

open Polynomial

theorem composition_of_polynomial_functions
  {R : Type} [CommRing R] (f g : R[X]) :
  ∃ (h : R[X]), ∀ (n : R), eval n h = eval (eval n g) f :=
begin
  induction f using Polynomial.induction_on',
  case h_add : f₁ f₂ h₁ h₂
  { obtain ⟨h₁', h₁_eq⟩ := h₁,
    obtain ⟨h₂', h₂_eq⟩ := h₂,
    use h₁' + h₂',
    intro n,
    simp [eval_add, h₁_eq, h₂_eq], },
  case h_monomial : a m
  { induction g using Polynomial.induction_on',
    case h_add : g₁ g₂ hg₁ hg₂
    { obtain ⟨h₁', h₁_eq⟩ := hg₁,
      obtain ⟨h₂', h₂_eq⟩ := hg₂,
      use a * (h₁' * h₂') ^ m,
      intro n,
      simp [eval_add, h₁_eq, h₂_eq, eval_monomial],
      ring, },
    case h_monomial : b p
    { use a * b ^ m * X ^ (m * p),
      intro n,
      simp [eval_monomial],
      ring, }, },
end

Note: This proof uses the `Polynomial.induction_on'` tactic to perform induction on the polynomials `f` and `g`. The base case is when `f` is a monomial, and the inductive step is when `f` is a sum of two polynomials. The same approach is used for `g`. The `use` tactic is used to specify the witness for the existential quantifier, and the `simp` and `ring` tactics are used to simplify and manipulate the polynomial expressions.
```
