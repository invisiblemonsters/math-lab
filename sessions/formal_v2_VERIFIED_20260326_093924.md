**Natural Language:**
THEOREM: The composition of two polynomial-time computable functions is itself polynomial-time computable.

PROOF SKETCH:
1. **Assume Functions**: Let `f : Σ* → Σ*` and `g : Σ* → Σ*` be two polynomial-time computable functions.
2. **Polynomial Bounds**: By definition, there exist polynomials `p`, `q` such that for all inputs `x`, the time complexity of `f(x)` is bounded by `p(|x|)` and `g(x)` by `q(|x|)`.
3. **Compose Functions**: Define `h(x) = g(f(x))`. We need to show `h` is polynomial-time computable.
4. **Time Complexity of h**: The time to compute `h(x)` is the time to compute `f(x)` plus the time to compute `g` on the output of `f(x)`.
5. **Polynomial Bound for h**: Since `|f(x)| ≤ p(|x|)` (as `f` runs in polynomial time, its output size is polynomially bounded), the time to compute `g(f(x))` is `q(p(|x|))`.
6. **Sum of Polynomials**: The total time for `h(x)` is `p(|x|) + q(p(|x|))`, which is a polynomial in `|x|`.
7. **Conclusion**: Thus, `h` is polynomial-time computable.

LEAN HINT:
- Import `complexity.basic` for polynomial-time computable definitions.
- Use `exists.intro` to provide the polynomial bound for `h`.
- Apply `polynomial.comp` and `polynomial.add` to construct the bounding polynomial.

**Lean 4 Proof:**

4. **Halting Guarantee**: For each input string, the decider for L* will halt because it only needs to check a finite number of decompositions, each of which is checked by the decider for L, which itself halts.
