**Natural Language:**
THEOREM: For any finite set A, the union of A with the empty set is equal to A itself.

PROOF SKETCH: 
1. Recall the definition of set union: `A ∪ B = {x | x ∈ A ∨ x ∈ B}`.
2. Substitute the empty set for B: `A ∪ ∅ = {x | x ∈ A ∨ x ∈ ∅}`.
3. Since `x ∈ ∅` is always false, the union simplifies to `{x | x ∈ A}`, which is just A.
4. Use the axiom of extensionality to conclude equality.

LEAN HINT:
```lean
import Data.Set.Basic

theorem union_empty (A : Set) : A ∪ ∅ = A := by
  simp [Set.union, Set.empty]
```

DIFFICULTY: easy

**Lean 4 Proof:**
```lean
import Mathlib

theorem union_empty (A : Set α) : A ∪ ∅ = A := 
  Set.ext (fun x => by simp [Set.mem_union, Set.mem_empty])
```

**Natural Language:**
THEOREM: For any finite set A, the intersection of A with itself is equal to A itself.

PROOF SKETCH:
1. Recall the definition of set intersection: A ∩ B = {x | x ∈ A ∧ x ∈ B}.
2. Substitute B with A: A ∩ A = {x | x ∈ A ∧ x ∈ A}.
3. Simplify the condition using the idempotent law of conjunction: x ∈ A ∧ x ∈ A ⇔ x ∈ A.
4. Conclude that A ∩ A = {x | x ∈ A} = A.

LEAN HINT:
- Imports: import set_theory.set.basic
- Tactics: simp, refl, ext (for set extensionality)

DIFFICULTY: easy

**Lean 4 Proof:**

import Mathlib

theorem intersection_idempotent {α : Type} (A : Set α) : A ∩ A = A := 
  by ext x; simp [Set.mem_inter]; tauto

**Natural Language:**
THEOREM: The intersection of two finite sets A and B, where A is a subset of B, is equal to A itself.

PROOF SKETCH:
1. Start with the definition of intersection: A ∩ B = {x | x ∈ A ∧ x ∈ B}.
2. Since A is a subset of B, for all x ∈ A, x is also ∈ B.
3. Therefore, for all x ∈ A, x ∈ A ∧ x ∈ B is always true.
4. Hence, A ∩ B = {x | x ∈ A} = A.

LEAN HINT:
- Imports: `import data.set.basic`
- Tactics: `intro`, `ext`, `simp`, `exact`

DIFFICULTY: easy

**Lean 4 Proof:**
```lean
import Mathlib

theorem intersection_subset (A B : Set α) (h : A ⊆ B) : A ∩ B = A := 
  Set.ext (λ x, 
    ⟨λ h', h'.1, λ h', ⟨h', h h'⟩⟩)
```

## Turn 5 — Verified ✓

**Natural Language:**
THEOREM: The composition of two polynomial-time computable functions is itself a polynomial-time computable function, and the resulting polynomial is the product of the individual polynomials.

PROOF SKETCH:
1. **Definitions**: Recall that a function f is polynomial-time computable if there exists a polynomial p such that f(x) can be computed in time p(|x|), where |x| denotes the length of input x.
2. **Composition**: Consider two polynomial-time computable functions f and g with polynomials p_f and p_g, respectively. We need to show that the composition h(x) = f(g(x)) is polynomial-time computable.
3. **Time Complexity**: The time to compute g(x) is bounded by p_g(|x|). Then, the time to compute f(g(x)) is bounded by p_f(|g(x)|). Since |g(x)| ≤ p_g(|x|) (as the output size of a polynomial-time function is polynomially bounded), we have p_f(p_g(|x|)), which is a polynomial in |x|.
4. **Polynomial Product**: The resulting polynomial is the composition p_f ∘ p_g, which is a polynomial (specifically, the product of the degrees of p_f and p_g plus the sum of their coefficients, but more accurately, the composition of polynomials is a polynomial).

LEAN HINT:
- Import necessary modules: `import Data.Polynomial` and `import Complexity.Basic`
- Define polynomial-time computable functions using existing structures (e.g., `PolynomialTimeComputable`).
- Use `apply` with `composition_polynomial` to establish the polynomial bound.
- Tactics: `simp`, `apply`, `exact`, `ring` for polynomial manipulations.

DIFFICULTY: Medium

**Lean 4 Proof:**
```lean
import Mathlib
import Data.Polynomial

open Polynomial

structure PolynomialTimeComputable (f : ℕ → ℕ) (p : Polynomial ℕ) where
  bound : ∀ x, f x ≤ p (x.natSize)

noncomputable def natSize (x : ℕ) : ℕ := x

theorem composition_polynomial_time_computable (f g : ℕ → ℕ) (p_f p_g : Polynomial ℕ)
  (h_f : PolynomialTimeComputable f p_f) (h_g : PolynomialTimeComputable g p_g) :
  PolynomialTimeComputable (f ∘ g) (p_f ∘ p_g) :=
begin
  apply PolynomialTimeComputable.mk,
  intro x,
  have g_bound : g x ≤ p_g (x.natSize) := h_g.bound x,
  have f_bound : f (g x) ≤ p_f ((g x).natSize) := h_f.bound (g x),
  have size_bound : (g x).natSize ≤ p_g (x.natSize) := by omega,
  have final_bound : f (g x) ≤ p_f (p_g (x.natSize)) :=
    by apply le_trans f_bound; apply p_f.mono; exact size_bound,
  exact final_bound,
end

theorem composition_polynomial_is_product (p_f p_g : Polynomial ℕ) :
  (p_f ∘ p_g).degree = p_f.degree * p_g.degree :=
by simp [Polynomial.degree_comp]

theorem composition_polynomial_coefficients (p_f p_g : Polynomial ℕ) :
  (p_f ∘ p_g).coeffs = p_f.coeffs.map (fun c => c * p_g.coeffs.prod) :=
by simp [Polynomial.coeffs_comp]
```

**Natural Language:**
THEOREM: The sum of two polynomial-time computable functions is itself a polynomial-time computable function, with the resulting polynomial being the maximum of the two individual polynomials.

PROOF SKETCH: 
1. **Define Polynomial-Time Computable Functions**: Let f and g be two functions that are computable in polynomial time with respective polynomials p and q.
2. **Sum Function Definition**: Define the sum function h(x) = f(x) + g(x).
3. **Computation Time Analysis**: The time to compute h(x) is the sum of the times to compute f(x) and g(x) individually. Since both f and g are computable in polynomial time, the maximum of p(|x|) and q(|x|) bounds the computation time for h(x).
4. **Polynomial Bound**: Show that max(p(|x|), q(|x|)) is a polynomial in |x|, ensuring h is polynomial-time computable.

LEAN HINT:
- Imports: `import Data.Polynomial` and relevant complexity theory definitions.
- Tactics: `apply`, `use` with the max polynomial, `simp` for algebraic manipulation, `exact` to confirm polynomial bounds.

DIFFICULTY: Medium

**Lean 4 Proof:**
```lean
import Mathlib
import Data.Polynomial

-- Define polynomial-time computable functions
def PolyTimeComputable (f : ℕ → ℕ) : Prop :=
  ∃ p : Polynomial ℕ, ∀ x, f x = p.eval (x : ℕ)

-- Theorem statement
theorem sum_poly_time_computable (f g : ℕ → ℕ)
  (hf : PolyTimeComputable f) (hg : PolyTimeComputable g) :
  PolyTimeComputable (λ x => f x + g x) :=
begin
  -- Extract polynomials for f and g
  obtain ⟨p, hp⟩ := hf
  obtain ⟨q, hq⟩ := hg

  -- Define the sum function
  let h : ℕ → ℕ := λ x => f x + g x

  -- Define the max polynomial
  let r : Polynomial ℕ := max p q

  -- Show that the max polynomial bounds the computation time for h
  use r
  intro x
  simp only [r, h, hp, hq]
  apply le_trans (add_le_add (p.eval x) (q.eval x) (max p q).eval x (max p q).eval x)
  exact le_max_of_le_left (p.eval x) (q.eval x)
end
```
