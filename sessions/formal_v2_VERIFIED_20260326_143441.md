THEOREM: The composition of a polynomial-time function with a polynomial-time computable function results in a polynomial-time computable function. Formally, if f: Σ* → Σ* is polynomial-time computable and g: Σ* → Σ* is polynomial-time computable, then the composition g ∘ f: Σ* → Σ* defined by (g ∘ f)(x) = g(f(x)) is also polynomial-time computable.

PROOF SKETCH:
1. **Assumptions**: Both f and g are polynomial-time computable, meaning there exist polynomials p and q such that the time complexity of f is bounded by p(|x|) and the time complexity of g is bounded by q(|x|), where |x| denotes the length of input x.
2. **Composition Time Complexity**: The time complexity of the composition g ∘ f is the sum of the time complexities of f and g. Specifically, for input x, the time taken by g ∘ f is the time taken by f(x) plus the time taken by g(f(x)).
3. **Polynomial Bound**: Since |f(x)| ≤ p(|x|) (as f is polynomial-time computable), the time complexity of g(f(x)) is bounded by q(p(|x|)). Therefore, the total time complexity of g ∘ f is bounded by p(|x|) + q(p(|x|)), which is a polynomial in |x|.
4. **Conclusion**: Hence, g ∘ f is polynomial-time computable.

**Natural Language:**
THEOREM: The composition of a polynomial-time computable function with a polynomial-time computable function is itself a polynomial-time computable function. Formally, if f : Σ* → Σ* is polynomial-time computable and g : Σ* → Σ* is polynomial-time computable, then the composition g ∘ f : Σ* → Σ* is also polynomial-time computable.

PROOF SKETCH:
1. **Assumption**: Both f and g are polynomial-time computable, meaning there exist polynomials p, q such that the time complexity of f is bounded by p and the time complexity of g is bounded by q.
2. **Composition Time Complexity**: The time complexity of g ∘ f is the sum of the time taken to compute f(x) and the time taken to compute g(f(x)). Since the input size to g is the output size of f(x), we need to bound this by a polynomial in the input size to f.
3. **Polynomial Bound**: Given that the output size of f(x) is polynomial in |x| (by the definition of polynomial-time computable functions), we can construct a polynomial r(|x|) = p(|x|) + q(p(|x|)), which bounds the total time complexity of g ∘ f.
4. **Conclusion**: Therefore, g ∘ f is polynomial-time computable.

LEAN HINT:
- Imports: `import Complexity.Basic`
- Tactics: `apply`, `use`, `have`, `simp`, `ring` for polynomial manipulation
- Key Definitions: `polynomial_time_computable`, `composition`

DIFFICULTY: medium

**Lean 4 Proof:**
```lean

import Complexity.Basic

open Complexity

theorem composition_polynomial_time_computable
  {f g : Σ* → Σ*} (hf : polynomial_time_computable f) (hg : polynomial_time_computable g) :
  polynomial_time_computable (g ∘ f) :=
begin
  obtain ⟨p, hp⟩ := hf,
  obtain ⟨q, hq⟩ := hg,
  use fun x => p x + q (p x),
  intro x,
  have h1 : (g ∘ f) x = g (f x) := rfl,
  have h2 : time_complexity (g ∘ f) x = time_complexity f x + time_complexity g (f x) := rfl,
  have h3 : time_complexity f x ≤ p x := hp x,
  have h4 : time_complexity g (f x) ≤ q (p x) := hq (f x),
  have h5 : output_size f x ≤ p x := hp x,
  simp [h1, h2],
  linarith [h3, h4],
  ring,
end
```
