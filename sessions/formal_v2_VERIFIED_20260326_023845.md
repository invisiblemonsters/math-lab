**Natural Language:**
THEOREM: For all natural numbers a, b, and c, if a + b = c, then the sum of the squares of a and b is less than or equal to the square of c plus the product of a and b, i.e., a² + b² ≤ c² + ab.

PROOF SKETCH:
1. Start with the given equation a + b = c.
2. Square both sides to obtain (a + b)² = c², which expands to a² + 2ab + b² = c².
3. Rearrange the equation to isolate a² + b², yielding a² + b² = c² - 2ab.
4. Notice that -2ab ≤ ab (since -2ab - ab = -3ab ≤ 0 for all natural numbers a, b), thus c² - 2ab ≤ c² + ab.
5. Conclude that a² + b² ≤ c² + ab by substitution.

LEAN HINT:
- Imports: `import Data.Nat`
- Tactics: `simp`, `omega`, `linarith`
- Key Steps: Use `have` to track inequalities, apply `linarith` for automatic inequality proving.

DIFFICULTY: medium

**Lean 4 Proof:**
```lean
import Mathlib

theorem sum_of_squares_inequality (a b c : Nat) (h : a + b = c) : a^2 + b^2 ≤ c^2 + a*b := 
  by have h1 : (a + b)^2 = c^2 := by rw [h]; ring
     have h2 : a^2 + 2*a*b + b^2 = c^2 := by simp [Nat.pow_add, mul_comm] at h1; assumption
     have h3 : a^2 + b^2 = c^2 - 2*a*b := by simp [h2]
     have h4 : 0 ≤ 3*a*b := by positivity
     have h5 : c^2 - 2*a*b ≤ c^2 + a*b := by linarith
     linarith [h3, h5]
```

**Natural Language:**
THEOREM: The composition of two polynomial-time computable functions is itself a polynomial-time computable function.

PROOF SKETCH: 
1. **Definitions**: Let f and g be polynomial-time computable functions. By definition, there exist polynomials p and q such that the time complexity of f is bounded by p and the time complexity of g is bounded by q.
2. **Composition Time Complexity**: The composition function h(x) = f(g(x)) requires time to compute g(x) plus the time to compute f on the result. If the input size is |x|, then |g(x)| is bounded by some polynomial in |x| (since g is polynomial-time computable).
3. **Polynomial Bound**: The time to compute h(x) is thus bounded by q(|x|) + p(q(|x|)), which is a polynomial in |x| because the sum and composition of polynomials are polynomials.
4. **Conclusion**: Therefore, h is polynomial-time computable.

LEAN HINT:
- Imports: `import Complexity.Basic`
- Tactics: `apply`, `simp`, `ring` for polynomial manipulations, `exact` to supply polynomial bounds.

DIFFICULTY: medium

**Lean 4 Proof:**

## Turn 10 — Verified ✓

**Natural Language:**
THEOREM: The composition of a polynomial-time computable function with a polynomial-time decidable language results in a polynomial-time decidable language.

PROOF SKETCH:
1. **Definitions**: Recall that a function f is polynomial-time computable if there exists a Turing machine that computes f in time polynomial in the input size. A language L is polynomial-time decidable if there exists a Turing machine that decides L in polynomial time.
2. **Composition Setup**: Consider a polynomial-time computable function f: Σ* → Σ* and a polynomial-time decidable language L. We need to show that the language {x | f(x) ∈ L} is polynomial-time decidable.
3. **Decision Procedure**: Construct a Turing machine that, on input x, computes f(x) and then checks if f(x) ∈ L. Since f is polynomial-time computable and L is polynomial-time decidable, the composition of these two steps runs in polynomial time.
4. **Polynomial Time Bound**: Let p(n) be the polynomial time bound for f, and q(n) be the polynomial time bound for L. The total time for the composed machine is p(n) + q(p(n)), which is still a polynomial in n.

LEAN HINT:
- Imports: `import complexity.basic`
- Tactics: `apply`, `intro`, `exact`, `simp` for polynomial composition, `use` for constructing the Turing machine.

DIFFICULTY: medium

**Lean 4 Proof:**
```lean
import Mathlib
import complexity.basic

open complexity

theorem composition_polynomial_time_decidable
  {Σ : Type} [fintype Σ]
  (f : Σ* → Σ*)
  (L : set Σ*)
  (f_poly : poly_time_computable f)
  (L_poly : poly_time_decidable L) :
  poly_time_decidable {x | f x ∈ L} :=
begin
  obtain ⟨M, p, hM⟩ from f_poly,
  obtain ⟨N, q, hN⟩ from L_poly,
  use (λ x, N (M x)),
  use (λ n, p n + q (p n)),
  intro x,
  simp only [poly_time_decidable, set.mem_set_of_eq],
  split,
  { intro hx,
    exact hN (f x) (hM x) hx, },
  { intro hx,
    obtain ⟨y, hy⟩ from hM x,
    obtain ⟨z, hz⟩ from hN (f x) hy hx,
    use z,
    exact hz, },
  intro n,
  apply le_trans _ (add_le_add (p n) (q (p n))),
  apply le_trans _ (p n + q (p n)),
  apply le_of_lt,
  simp only [poly_time_computable, poly_time_decidable],
  exact hM x n,
end
```
