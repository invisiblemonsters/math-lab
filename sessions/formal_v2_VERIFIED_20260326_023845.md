**Natural Language:**
THEOREM: For all natural numbers a, b, and c, if a + b = c, then the sum of the squares of a and b is less than or equal to the square of c plus the product of a and b, i.e., a² + b² ≤ c² + ab.

PROOF SKETCH:
1. Start with the given equation a + b = c.
2. Square both sides to obtain (a + b)² = c², which expands to a² + 2ab + b² = c².
3. Rearrange the equation to isolate a² + b², yielding a² + b² = c² - 2ab.
4. Notice that -2ab ≤ ab (since -2ab - ab = -3ab ≤ 0 for all natural numbers a, b), thus c² - 2ab ≤ c² + ab.
5. Conclude that a² + b² ≤ c² + ab by substitution.

LEAN HINT:
- Imports: `import Data.Nat`
- Tactics: `simp`, `omega`, `linarith`
- Key Steps: Use `have` to track inequalities, apply `linarith` for automatic inequality proving.

DIFFICULTY: medium

**Lean 4 Proof:**
```lean
import Mathlib

theorem sum_of_squares_inequality (a b c : Nat) (h : a + b = c) : a^2 + b^2 ≤ c^2 + a*b := 
  by have h1 : (a + b)^2 = c^2 := by rw [h]; ring
     have h2 : a^2 + 2*a*b + b^2 = c^2 := by simp [Nat.pow_add, mul_comm] at h1; assumption
     have h3 : a^2 + b^2 = c^2 - 2*a*b := by simp [h2]
     have h4 : 0 ≤ 3*a*b := by positivity
     have h5 : c^2 - 2*a*b ≤ c^2 + a*b := by linarith
     linarith [h3, h5]
```

## Turn 8 — Verified ✓

**Natural Language:**
THEOREM: The composition of two polynomial-time computable functions is itself a polynomial-time computable function.

PROOF SKETCH: 
1. **Definitions**: Let f and g be polynomial-time computable functions. By definition, there exist polynomials p and q such that the time complexity of f is bounded by p and the time complexity of g is bounded by q.
2. **Composition Time Complexity**: The composition function h(x) = f(g(x)) requires time to compute g(x) plus the time to compute f on the result. If the input size is |x|, then |g(x)| is bounded by some polynomial in |x| (since g is polynomial-time computable).
3. **Polynomial Bound**: The time to compute h(x) is thus bounded by q(|x|) + p(q(|x|)), which is a polynomial in |x| because the sum and composition of polynomials are polynomials.
4. **Conclusion**: Therefore, h is polynomial-time computable.

LEAN HINT:
- Imports: `import Complexity.Basic`
- Tactics: `apply`, `simp`, `ring` for polynomial manipulations, `exact` to supply polynomial bounds.

DIFFICULTY: medium

**Lean 4 Proof:**

```lean
import Complexity.Basic

open Complexity

theorem composition_polynomial_time {α β γ : Type} [SizeOf α] [SizeOf β] [SizeOf γ]
  (f : β → γ) (g : α → β) (p : ℕ → ℕ) (q : ℕ → ℕ) (hp : IsPolynomial p) (hq : IsPolynomial q)
  (hf : PolynomialTime f p) (hg : PolynomialTime g q) :
  PolynomialTime (f ∘ g) (fun n => q n + p (q n)) :=
begin
  apply PolynomialTime.bind (fun x => g x) (fun y => f y),
  exact hg,
  apply PolynomialTime.map (fun y => f y),
  intro y,
  apply hf,
  apply IsPolynomial.add hq (IsPolynomial.comp hp hq),
  simp [Function.comp],
  exact rfl,
end
```
