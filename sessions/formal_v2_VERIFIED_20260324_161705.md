THEOREM: The set of decidable languages is closed under the Kleene star operation.

PROOF SKETCH: 
1. **Definition Recall**: A language L is decidable if there exists a Turing machine that halts and accepts or rejects any input string. The Kleene star of a language L, denoted L*, is the set of all finite strings formed by concatenating zero or more strings from L.
2. **Turing Machine Construction**: To show L* is decidable, construct a Turing machine that simulates the decider for L on all possible prefixes of the input string, verifying that each prefix is in L and ensuring the entire string can be segmented into strings from L.
3. **Decidability Verification**: Demonstrate that this Turing machine always halts by noting that the input string has finite length, and the number of possible segmentations, though exponential, does not affect the halting condition due to the finite verification process for each prefix.

LEAN HINT:
- Imports: `import computability.turing_machine`
- Tactics: `apply`, `use` (to provide the constructed Turing machine), `intro` (to generalize over input strings), `cases` (to handle different segmentations)

DIFFICULTY: Medium

**Lean 4 Proof:**
```lean
import Mathlib
import computability.turing_machine

-- Define decidability of a language
def decidable_language (L : set (list ℕ)) : Prop :=
  ∃ (M : turing_machine), ∀ (x : list ℕ), 
    (M.eval x).is_some ∧ ((M.eval x).get = 0 ∨ (M.eval x).get = 1)

-- Kleene star operation
def kleene_star (L : set (list ℕ)) : set (list ℕ) :=
  { x | ∃ (n : ℕ) (l : list (list ℕ)), 
    x = l.concat ∧ l.length = n ∧ ∀ (i : ℕ) (h : i < n), 
    (l.nth_le i h) ∈ L }

-- Theorem statement
theorem decidable_language_kleene_star 
  (L : set (list ℕ)) (h : decidable_language L) : 
  decidable_language (kleene_star L) := 
begin
  -- Extract the Turing machine for L
  rcases h with ⟨M, hM⟩,

  -- Construct a new Turing machine for L*
  use (λ x, 
    let n := x.length in 
    let l := (list.range n).map (λ i, x.take (i + 1)) in 
    let evals := l.map (λ prefix, M.eval prefix) in 
    if ∀ (e : option ℕ) (he : e ∈ evals), e.is_some ∧ (e.get = 0 ∨ e.get = 1) 
    then some 0 else some 1),

  -- Verify the new Turing machine works for all inputs
  intro x,

  -- Consider all possible segmentations of the input string
  cases h' : ∃ (n : ℕ) (l : list (list ℕ)), 
    x = l.concat ∧ l.length = n ∧ ∀ (i : ℕ) (h : i < n), 
    (l.nth_le i h) ∈ L,
  { -- If x can be segmented into strings from L
    simp [kleene_star, h'],
    use 0,
    simp [hM],
    omega },
  { -- If x cannot be segmented into strings from L
    simp [kleene_star, h'],
    use 1,
    simp [hM],
    omega }
end
```

**Natural Language:**
THEOREM: The set of decidable languages is closed under the operation of homomorphism. Specifically, if a language L is decidable and a function h: Σ → Γ* is a homomorphism on strings, then the language h(L) = { h(w) | w ∈ L } is also decidable.

PROOF SKETCH:
1. **Definitions**: Recall that a homomorphism h extends to strings by h(w) = h(w₁)h(w₂)...h(wₙ) for w = w₁w₂...wₙ.
2. **Decidability of L**: Since L is decidable, there exists a Turing machine M that halts and accepts/rejects on input w ∈ Σ*.
3. **Constructing M'**: Build a new Turing machine M' that, on input v ∈ Γ*, checks if v can be "parsed" as h(w) for some w ∈ Σ*. This involves nondeterministically guessing a decomposition of v into parts corresponding to h(σ) for each σ ∈ Σ.
4. **Simulation**: For each guessed decomposition, simulate M on the corresponding w. If M accepts w, then M' accepts v.
5. **Decidability of h(L)**: Since M' can effectively decide membership in h(L) by exploring all possible decompositions (which are polynomially bounded if h maps symbols to strings of fixed length), h(L) is decidable.

LEAN HINT:
- Imports: `import computability.turing_machine`
- Tactics: `apply`, `use` (for constructing M'), `simp` (for verifying the homomorphism property), `induction` (on string length for parsing)

DIFFICULTY: medium

**Lean 4 Proof:**
```lean
import Mathlib
import computability.turing_machine

-- Define the homomorphism on strings
def homomorphism (Σ Γ : Type) [Fintype Σ] [Fintype Γ] (h : Σ → List Γ) : List Σ → List Γ
| [] => []
| σ :: w => h σ ++ homomorphism h w

-- Theorem statement
theorem decidability_closed_under_homomorphism (Σ Γ : Type) [Fintype Σ] [Fintype Γ]
  (L : Set (List Σ)) (h : Σ → List Γ) (M : TuringMachine Σ Bool)
  (M_decides_L : ∀ w, M.eval w = (w ∈ L)) :
  ∃ M' : TuringMachine Γ Bool, ∀ v, M'.eval v = (v ∈ Set.image (homomorphism h) L) :=
begin
  -- Constructing M'
  let M' : TuringMachine Γ Bool := {
    next := fun (q : M.States) (v : Γ) =>
      match q with
      | M.start =>
        -- Nondeterministically guess a decomposition of v
        { q' := M.start, tape := v :: M.tape, result := M.result } ∨
        { q' := M.start, tape := [] :: M.tape, result := M.result }
      | M.loop σ =>
        -- Simulate M on the corresponding w
        match M.next (M.loop σ) σ with
        | { q' := M.halt b, .. } =>
          { q' := M.halt b, tape := M.tape, result := b }
        | { q' := q', .. } =>
          { q' := M.loop q', tape := M.tape, result := M.result }
      | M.halt b =>
        { q' := M.halt b, tape := M.tape, result := b }
    ,
    start := M.start,
    halt := fun q => q = M.halt true ∨ q = M.halt false,
    result := fun q => q = M.halt true
  }

  -- Verify the homomorphism property
  have homomorphism_property : ∀ w, homomorphism h w = List.bind w h :=
  begin
    intro w
    induction w
    case nil =>
      simp [homomorphism]
    case cons σ w ih =>
      simp [homomorphism, ih]
  end

  -- Decidability of h(L)
  apply funext
  intro v
  simp [M'.eval, homomorphism_property]
  apply propext
  split
  case h.mp =>
    intro h_v
    use List.bind v fun γ => List.pmap (fun σ => h σ) (List.pmap (fun γ' => γ') γ)
    simp [List.bind, List.pmap]
    apply List.exists_imp_exists
    intro w h_w
    simp [h_w]
    apply M_decides_L
  case h.mpr =>
    intro h_v
    apply List.exists_imp_exists
    intro w h_w
    simp [h_w]
    apply M_decides_L
end
```
