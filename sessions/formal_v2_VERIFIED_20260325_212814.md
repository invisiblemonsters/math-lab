**Natural Language:**
THEOREM: For any two finite sets A and B, the cardinality of their symmetric difference is equal to the sum of the cardinalities of A and B minus twice the cardinality of their intersection.

PROOF SKETCH:
1. **Definition Recall**: The symmetric difference of A and B is defined as (A \ B) ∪ (B \ A).
2. **Cardinality Expansion**: Use the property that |A ∪ B| = |A| + |B| - |A ∩ B|, adapting it for symmetric difference.
3. **Set Disjointness**: Note that (A \ B) and (B \ A) are disjoint, so |(A \ B) ∪ (B \ A)| = |A \ B| + |B \ A|.
4. **Simplify**: Express |A \ B| as |A| - |A ∩ B| and |B \ A| as |B| - |A ∩ B|, then combine terms.

LEAN HINT:
- Imports: `import data.set.basic`
- Tactics: `simp`, `rw` (rewrite) using set cardinality lemmas from Mathlib (e.g., `set.card_union`, `set.card_diff`).

DIFFICULTY: medium

**Lean 4 Proof:**
```lean
import Mathlib
import data.set.basic

open Set

theorem symmetric_difference_cardinality (A B : Set ℕ) (hA : A.Finite) (hB : B.Finite) :
  (A Δ B).card = A.card + B.card - 2 * (A ∩ B).card :=
begin
  simp only [symm_diff_def],
  have h_disjoint : Disjoint (A \ B) (B \ A) := disjoint_diff_diff,
  rw [card_union_eq h_disjoint, card_diff hA, card_diff hB],
  simp [add_assoc, add_comm],
  rw [sub_eq_add_neg, add_assoc, add_comm],
  simp [mul_comm, mul_neg],
  apply rfl,
end
```

**Natural Language:**
THEOREM: For all natural numbers a, b, and c, if a + b = c, then the maximum of a, b, and c is equal to the maximum of a and b.

PROOF SKETCH:
1. **Assume a + b = c**: Start with the given equation.
2. **Case analysis on a and b**:
   - If a ≥ b, then max(a, b) = a. Since a + b = c, we have c = a + b ≥ a (because b is non-negative). Thus, max(a, b, c) = max(a, c) = c, but since a + b = c and a ≥ b, c could be larger than a, so we need to show max(a, b, c) = max(a, b) in this context.
   - Similarly, if b ≥ a, max(a, b) = b, and c = a + b ≥ b, leading to max(a, b, c) = max(b, c) = c. However, the goal is to show this equals max(a, b), which requires demonstrating that c, being the sum, doesn't exceed the larger of a or b in a way that affects the maximum when considering the original equation.
3. **Resolution**:
   - Recognize that if a + b = c, then c is at least as large as both a and b individually because both a and b are non-negative. This implies that max(a, b, c) = c.
   - However, the theorem statement claims max(a, b, c) = max(a, b). This suggests a mistake in the initial intuition. Correct approach:
     - Since c = a + b, and we know that max(a, b) ≤ a + b (because either a or b is the maximum, and adding two positives will always be greater than or equal to the maximum of the two), thus max(a, b, c) = c. But the theorem as stated seems to suggest otherwise, indicating a potential error in the theorem statement itself.
     - **Correction to Proof Sketch**:
       - Realize the theorem as stated might not hold. For example, let a = 1, b = 2, then c = 3. max(a, b, c) = 3, but max(a, b) = 2. Thus, the original theorem statement is incorrect.
       - **Revised Theorem**: For all natural numbers a, b, and c, if a + b = c, then the maximum of a, b, and c is equal to c.
       - **Proof**:
         1. From a + b = c, since a, b are natural numbers, c ≥ a and c ≥ b by definition of addition.
         2. Therefore, max(a, b, c) = c.
