## Turn 1 — Verified ✓

**Natural Language:**
THEOREM: The sum of the first n natural numbers, denoted as 1 + 2 + ... + n, is equal to n(n + 1)/2.

PROOF SKETCH: We will use mathematical induction to prove this statement. 
1. **Base Case**: For n = 1, the left-hand side (LHS) is 1, and the right-hand side (RHS) is 1(1 + 1)/2 = 1. Thus, the base case holds.
2. **Inductive Step**: Assume the statement is true for some arbitrary natural number k, i.e., 1 + 2 + ... + k = k(k + 1)/2. We need to show it holds for k + 1. 
   - Starting with the sum up to k + 1: 1 + 2 + ... + k + (k + 1).
   - By the inductive hypothesis, this equals k(k + 1)/2 + (k + 1).
   - Factoring out (k + 1) gives (k + 1)(k/2 + 1) = (k + 1)(k + 2)/2, which matches the formula for n = k + 1.

LEAN HINT:
```lean
import data.nat.basic

theorem sum_first_n_natural_numbers (n : ℕ) : 
  (list.sum (list.range (n + 1))) = n(n + 1)/2 :=
begin
  induction n,
  { simp [list.sum, list.range] }, -- Base case
  { simp [list.sum, list.range, *] } -- Inductive step
end
```

**Natural Language:**
THEOREM: The composition of two polynomial-time functions is also a polynomial-time function. Specifically, if f is a polynomial-time function from Σ* to Σ* and g is a polynomial-time function from Σ* to Σ*, then the function h defined by h(x) = f(g(x)) is also a polynomial-time function.

PROOF SKETCH: 
1. **Definitions**: Start with the definitions of polynomial-time functions f and g. 
2. **Composition**: Construct the composition function h(x) = f(g(x)).
3. **Time Complexity**: Use the fact that the time complexity of h is bounded by the sum of the time complexities of f and g, adjusted for the input size. Since both f and g have polynomial time complexities, their composition will also have a polynomial time complexity.
4. **Formalization**: Apply the polynomial composition lemma to show that the resulting time complexity is polynomial in the input size.

LEAN HINT:
```lean
import Complexity.Basic

theorem polynomial_time_composition 
  {f g : Σ* → Σ*} 
  (hf : polynomial_time f) 
  (hg : polynomial_time g) : 
  polynomial_time (λ x, f (g x)) :=
begin
  -- Unfold the definitions of polynomial_time for f and g
  cases hf, cases hg,
  -- Construct the composition and its time bound
  refine ⟨_, _⟩,
  -- Use polynomial arithmetic to show the bound is polynomial
  apply polynomial.sum_closure,
  -- Apply the time bounds of f and g
  exact hg.time_bound,
  -- Adjust for the input size transformation through g
  apply polynomial.comp_closure,
  exact hf.time_bound,
  exact hg.time_bound,
end
```

**Natural Language:**
THEOREM: The composition of a polynomial-time function with a polynomial-time decidable language is also a polynomial-time decidable language. Specifically, if f is a polynomial-time function from Σ* to Σ* and L is a language in P, then the language {w | f(w) ∈ L} is also in P.

PROOF SKETCH: 
1. **Assume** f is a polynomial-time function, meaning there exists a polynomial p such that for all inputs w, the computation time of f(w) is bounded by p(|w|).
2. **Assume** L is in P, meaning there exists a polynomial q and a Turing machine M that decides L in time bounded by q(|x|) for all x ∈ Σ*.
3. **Construct** a Turing machine M' that, on input w, computes f(w) and then simulates M on f(w).
4. **Show** that the running time of M' is bounded by a polynomial in |w|, specifically by the polynomial r(n) = p(n) + q(p(n)), which is still a polynomial.
5. **Conclude** that {w | f(w) ∈ L} is in P by demonstrating that M' decides it in polynomial time.

LEAN HINT:
```lean
import Complexity.Basic

theorem polynomial_composition_decidable (f : polynomial_time_function) (L : language) (hL : L ∈ P) :
  {w | f w ∈ L} ∈ P :=
begin
  -- Unfold definitions of polynomial_time_function and P
  rcases f with ⟨p, f_poly⟩,
  rcases hL with ⟨q, M, hM⟩,
  -- Construct the new Turing machine
  use [r := λn, p n + q (p n), M'],
  -- Verify the running time and correctness
  apply le_trans; [exact f_poly, exact hM],
  -- Composition of polynomials is a polynomial (invoke Mathlib's polynomial lemmas)
  exact polynomial.add_closure p q,
end
```
