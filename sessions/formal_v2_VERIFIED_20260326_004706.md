3. **Inductive Hypothesis Application**: Demonstrate that if the formula is true for k, it also holds for k + 1 by adding (k + 1) to both sides of the equation and simplifying to show it equals (k + 1)(k + 2)/2.
4. **Conclusion**: Conclude by induction that the formula is valid for all natural numbers n.

THEOREM: The union of two finite sets is finite, and its cardinality is at most the sum of the cardinalities of the individual sets.

PROOF SKETCH:

3. **Cardinality of Union**: The cardinality of the union |A ∪ B| is at most m + n, since every element in A ∪ B is either in A, in B, or in both.
4. **Inclusion-Exclusion Principle**: Use the principle to show |A ∪ B| = |A| + |B| - |A ∩ B|, which is ≤ |A| + |B|.

  -- Apply finite.union and use cardinality properties

THEOREM: The sum of the first n natural numbers starting from a non-zero value m is equal to (n/2) * (2m + (n - 1)), i.e., m + (m+1) + ... + (m + n - 1) = n * (2m + n - 1) / 2.

PROOF SKETCH:
1. **Base Case**: Verify the formula for n = 1, showing that m = (1/2) * (2m + 1 - 1) simplifies correctly.
2. **Inductive Step**: Assume the formula holds for some k, then prove it for k + 1 by adding the next term (m + k) to both sides and simplifying.
3. **Algebraic Manipulation**: Use basic arithmetic operations and algebraic properties to rearrange and confirm the equality.

LEAN HINT:
- Imports: `import data.nat.basic`
- Tactics: `induction n`, `simp`, `ring`, `exact`, `omega`

DIFFICULTY: medium

**Lean 4 Proof:**
```lean
import Mathlib
import data.nat.basic

theorem sum_of_first_n_natural_numbers (m n : ℕ) : 
  (List.range n).map (λ i => m + i).foldl (λ a b => a + b) 0 = n * (2 * m + n - 1) / 2 :=
begin
  induction n with k hk,
  { simp, exact Nat.zero_mul _ },
  { simp only [List.range_succ, List.map_append, List.foldl_append],
    simp only [List.foldl_nil, List.foldl_cons],
    have : k * (2 * m + k - 1) / 2 + (m + k) = (k + 1) * (2 * m + (k + 1) - 1) / 2,
    { ring },
    exact this ▸ (congr_arg (λ x => x + (m + k)) hk) }
end
However, the above solution uses List.range, List.map and List.foldl which are not very efficient for this problem. Here is a more straightforward solution using mathematical induction:

import Mathlib
import data.nat.basic

theorem sum_of_first_n_natural_numbers (m n : ℕ) : 
  ∃ s : ℕ, (∀ i : ℕ, i < n → s = m + i + (∑ j in Finset.range i, m + j)) :=
begin
  induction n with k hk,
  { use m, intro i, intro hi, exfalso, exact Nat.not_lt_zero i hi },
  { obtain ⟨s, hs⟩ := hk,
    use s + (m + k),
    intro i, cases i,
    { simp },
    { intro hi,
      have : i < k := Nat.lt_of_lt_pred hi,
      simp [hs i this] } }
end

theorem sum_of_first_n_natural_numbers_eq (m n : ℕ) : 
  (∑ i in Finset.range n, m + i) = n * (2 * m + n - 1) / 2 :=
begin
  induction n with k hk,
  { simp },
  { have : (∑ i in Finset.range (k + 1), m + i) = (∑ i in Finset.range k, m + i) + (m + k),
    { simp only [Finset.sum_range_succ] },
    simp only [this, hk],
    ring }
end
```

**Natural Language:**
THEOREM: The composition of a polynomial-time computable function with a polynomial-time decidable language is itself polynomial-time decidable.

PROOF SKETCH: 
1. **Definitions**: Recall that a language L is polynomial-time decidable if there exists a polynomial-time computable function f such that f(x) = 1 if x ∈ L and f(x) = 0 otherwise. Similarly, a function g is polynomial-time computable if it can be computed in time bounded by a polynomial in the input size.
2. **Composition**: Consider the composition h(x) = f(g(x)), where f is the decision function for language L and g is a polynomial-time computable function.
3. **Polynomial-Time Bound**: Since both f and g are polynomial-time computable, there exist polynomials p and q such that the time complexity of f is bounded by p and the time complexity of g is bounded by q. The composition h(x) can be computed in time bounded by p(q(|x|)), which is still a polynomial in |x|.
4. **Decidability**: The language defined by h (i.e., {x | h(x) = 1}) is therefore polynomial-time decidable because h itself is a polynomial-time computable function.

- Define polynomial-time computable functions and languages using existing structures in Mathlib (e.g., `polynomial_time_computable`).
- Use `exists.intro` to provide the composed function and its polynomial bound.
- Apply `polynomial.comp` to establish the polynomial bound on the composition.

  (L : set string) 
  (f : string → ℕ) 
  (g : string → string) 
  (h : polynomial_time_computable f) 
  (p : polynomial ℕ)

  (hg : polynomial_time_computable g)

    (∀ x, h' x = 1 ↔ g x ∈ L) ∧ 
    (∀ x, h'.time_complexity x ≤ p' (string.length x)) :=

    λ x, by rw [hf, h'],
  exact exists.intro h' (exists.intro p' (and.intro h'_computable (and.intro h'_correct h'_bound))),

**Natural Language:**
THEOREM: The set of all decidable languages is closed under the operation of taking complements, i.e., if a language L is decidable, then its complement (the set of all strings not in L) is also decidable.

PROOF SKETCH:
1. **Definition of Decidable Language**: Recall that a language L is decidable if there exists a Turing machine (or an algorithm) that halts on all inputs and decides membership in L.
2. **Complement Construction**: Given a decider for L, construct a decider for the complement of L by running the original decider and inverting its output (accepting when the original decider rejects and vice versa).
3. **Correctness Argument**: Show that this construction correctly decides the complement language by leveraging the totality (halting on all inputs) of the original decider.
4. **Formalization in Lean**: Utilize existing definitions of Turing machines or oracle machines in Mathlib, and apply straightforward modification of the decider's transition function.

LEAN HINT:
- Imports: `import computability.turing_machine`
- Tactics: `apply`, `intro`, `cases`, `simp`, `exact`
- Key Definitions: `turing_machine.decider`, `turing_machine.complement`

DIFFICULTY: medium

theorem decidable_language_closed_under_complement
  {α : Type} [enum α] (L : set α) :

  intro d,
  apply turing_machine.complement,
  exact d,

  {α : Type} [enum α] (L : set α) (d : turing_machine.decider L) :
