**Natural Language:**
THEOREM: The sum of the first n natural numbers, denoted as 1 + 2 + ... + n, is equal to n(n + 1)/2.

PROOF SKETCH: 
1. **Base Case**: Verify the formula for n = 1, showing that 1 = 1(1 + 1)/2.
2. **Inductive Step**: Assume the formula holds for some arbitrary natural number k (i.e., 1 + 2 + ... + k = k(k + 1)/2).
3. **Inductive Hypothesis Application**: Demonstrate that if the formula is true for k, it also holds for k + 1 by adding (k + 1) to both sides of the equation and simplifying to show that it equals (k + 1)(k + 2)/2.
4. **Conclusion**: Conclude by induction that the formula is valid for all natural numbers n.

LEAN HINT: 
- Imports: `import data.nat.basic`
- Tactics: `induction n`, `simp`, `ring`, `exact`

DIFFICULTY: easy

**Lean 4 Proof:**
```lean
import Mathlib
import data.nat.basic

theorem sum_of_first_n_natural_numbers (n : ℕ) : 
  (list.sum (list.range (n + 1))) = n * (n + 1) / 2 :=
begin
  induction n with k ih,
  { simp, exact rfl },
  { simp [list.sum, list.range, list.append],
    have h : list.sum (list.range (k + 1)) = k * (k + 1) / 2 := ih,
    rw h,
    ring }
end
```

**Natural Language:**
THEOREM: For all natural numbers n and m, the minimum of n and m is less than or equal to the maximum of n and m, with equality if and only if n equals m.

PROOF SKETCH: 
1. **Definition of min and max**: Recall that min(n, m) = n if n ≤ m, else m; max(n, m) = m if n ≤ m, else n.
2. **Case analysis**: Consider two cases based on the ordering of n and m.
   - **Case 1**: If n ≤ m, then min(n, m) = n and max(n, m) = m. Since n ≤ m, we have n ≤ m, so min(n, m) ≤ max(n, m).
   - **Case 2**: If m < n, then min(n, m) = m and max(n, m) = n. Since m < n, we have m ≤ n, so min(n, m) ≤ max(n, m).
3. **Equality condition**: For the "if and only if" part, assume n = m. Then min(n, m) = max(n, m) = n = m. Conversely, if min(n, m) = max(n, m), then the case analysis shows n must equal m.

LEAN HINT:
```lean
import data.nat.basic

theorem min_le_max_eq_iff (n m : ℕ) : min n m ≤ max n m ∧ (min n m = max n m ↔ n = m) :=
begin
  { show min n m ≤ max n m, by { by_cases n ≤ m; simp [min, max, *]; refl } },
  { split; intro h,
    { by_cases n ≤ m; simp [min, max, *] at h; try { contradiction }, refl },
    { simp [h] } }
end
```

**Natural Language:**
THEOREM: For any natural number n, the sum of the first n odd numbers is equal to n². Formally, 1 + 3 + 5 + ... + (2n-1) = n².

PROOF SKETCH: 
1. **Base Case**: Verify the statement for n = 1, where the sum is 1 and 1² = 1.
2. **Inductive Step**: Assume the statement holds for some k (i.e., 1 + 3 + ... + (2k-1) = k²). Then, for k + 1, the sum becomes k² + (2(k+1)-1) = k² + 2k + 1 = (k + 1)².

LEAN HINT: 
- Imports: `import data.nat.basic`
- Tactics: `induction n with n ih`, `simp [pow_two, mul_add, add_mul]`, `norm_num`

DIFFICULTY: easy

**Lean 4 Proof:**
```lean
import Mathlib
import data.nat.basic

def sum_odd_numbers : Nat → Nat
| 0 => 0
| n + 1 => sum_odd_numbers n + 2*n + 1

theorem sum_odd_numbers_eq (n : Nat) : sum_odd_numbers n = n^2 :=
by induction n with n ih
· simp [pow_two]
· simp [sum_odd_numbers, pow_two, mul_add, add_mul]
  rw [ih]
  ring
```

**Natural Language:**
THEOREM: The composition of two polynomial-time computable functions is also polynomial-time computable. Formally, if f: Σ* → Σ* and g: Σ* → Σ* are polynomial-time computable functions, then the function h: Σ* → Σ* defined by h(x) = f(g(x)) is also polynomial-time computable.

PROOF SKETCH: 
1. **Definition of Polynomial-Time Computable**: Recall that a function f is polynomial-time computable if there exists a Turing machine M that computes f and runs in time bounded by a polynomial p(|x|) in the length of the input x.
2. **Composition Time Bound**: Let p_f and p_g be the polynomial time bounds for f and g, respectively. The time to compute h(x) = f(g(x)) is the sum of the time to compute g(x) and the time to compute f(g(x)). 
3. **Polynomial Addition**: Since the sum of two polynomials is also a polynomial, the total time to compute h(x) is bounded by a polynomial in |x|.

LEAN HINT:
- Import `complexity.basic` for polynomial-time computable definitions.
- Use `add_poly` lemma for polynomial addition.
- Apply `exists.intro` for the composed Turing machine.

DIFFICULTY: medium

**Lean 4 Proof:**
```lean
import Mathlib
import complexity.basic

open complexity

theorem composition_polynomial_time
  {f g : string → string}
  (hf : poly_time f)
  (hg : poly_time g) :
  poly_time (λ x => f (g x)) :=
begin
  cases hf with Mf p_f,
  cases hg with Mg p_g,
  apply exists.intro (Mg.seq Mf),
  apply exists.intro (add_poly p_g p_f),
  intro x,
  simp [poly_time, time_bound],
  apply le_trans (time_comp (Mg.seq Mf) x),
  apply le_trans (add_le_add (time_bound Mg p_g x) (time_bound Mf p_f (g x))),
  apply add_poly_spec,
end
```
