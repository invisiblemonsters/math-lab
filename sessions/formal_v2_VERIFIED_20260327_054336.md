THEOREM: Polynomial-time reductions are compatible with the Kleene star operation. Formally, if there exists a polynomial-time reduction from language L₁ to L₂, then there also exists a polynomial-time reduction from the Kleene star of L₁ (denoted L₁*) to the Kleene star of L₂ (denoted L₂*).

PROOF SKETCH: 
1. **Definition of Polynomial-time Reduction**: Recall that a polynomial-time reduction from L₁ to L₂ is a polynomial-time computable function f such that for all strings w, w ∈ L₁ if and only if f(w) ∈ L₂.
2. **Kleene Star Definition**: The Kleene star L₁* consists of all finite concatenations of strings from L₁, including the empty string.
3. **Construct Reduction for Kleene Star**: Given a string in L₁*, which is a concatenation of strings w₁, w₂, ..., wₙ where each wᵢ ∈ L₁, apply the reduction f to each wᵢ individually to obtain f(w₁), f(w₂), ..., f(wₙ).
4. **Concatenate Results**: Concatenate the results f(w₁), f(w₂), ..., f(wₙ) to form a new string in L₂*. Since each f(wᵢ) ∈ L₂, their concatenation is in L₂*.
5. **Polynomial-time Computability**: Show that the function mapping the concatenation of wᵢ's to the concatenation of f(wᵢ)'s is polynomial-time computable. This follows from the fact that f is polynomial-time and the concatenation of polynomially many polynomial-length strings remains polynomial-length.
6. **Equivalence Preservation**: Verify that the original string is in L₁* if and only if the constructed string is in L₂*. This follows directly from the properties of f and the definition of Kleene star.

```lean
begin
  -- Construct the reduction function for Kleene star
  refine ⟨λ s, (s.split_on L₁).map f⟩,
  -- Prove equivalence preservation
  intros s,
  split,
  { intro h, -- If s ∈ L₁*, then split into L₁ parts, map with f, and show in L₂*
    rw mem_kleene_star at h,
    induction h,
    case empty { exact mem_kleene_star.empty },
    case concat { w _ hw } { exact mem_kleene_star.concat (h_ih hw) } },
  { intro h, -- Conversely, if mapped string is in L₂*, invert the reduction
    rw mem_kleene_star at h,
    induction h,
    case empty { exact mem_kleene_star.empty },
    case concat { w _ hw } { exact mem_kleene_star.concat (h_ih hw) } }
```

**Natural Language:**
THEOREM: The complexity class P is closed under the intersection operation. Formally, if languages L₁ and L₂ over an alphabet Σ are in P, then the language L₁ ∩ L₂ = {w ∈ Σ* | w ∈ L₁ and w ∈ L₂} is also in P.

PROOF SKETCH: 
1. **Definition of P**: Recall that a language L is in P if there exists a polynomial-time decidable algorithm (e.g., a Turing machine) that decides membership in L.
2. **Decidability of Intersection**: Construct a polynomial-time decidable algorithm for L₁ ∩ L₂ by running the algorithms for L₁ and L₂ sequentially on input w and returning "accept" only if both algorithms accept.
3. **Polynomial-Time Bound**: Show that the running time of the constructed algorithm is bounded by the sum of the running times of the algorithms for L₁ and L₂, which is still polynomial in the length of the input.
4. **Formal Closure**: Conclude that since the intersection can be decided in polynomial time, L₁ ∩ L₂ is in P, thus establishing closure under intersection.

LEAN HINT:
```lean

theorem P_closed_under_intersection {Σ : Type} [fintype Σ] (L₁ L₂ : language Σ) (h₁ : L₁ ∈ P) (h₂ : L₂ ∈ P) : L₁ ∩ L₂ ∈ P :=
begin
  -- Unfold the definition of P and language intersection
  rw [mem_P, language.inter_eq],
  -- Construct the decider for the intersection
  use (λw, (h₁.decider w) && (h₂.decider w)),
  -- Prove the decider is polynomial-time
  apply polynomial_time.and,
  -- Apply existing polynomial-time properties of L₁ and L₂
  exact h₁.poly_time,
  exact h₂.poly_time,
end
```
